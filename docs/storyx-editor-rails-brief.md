# Story X 에디터 좌·우 레일 UI/UX 재디자인 의뢰

Story X는 한국어 장편 소설·에세이 창작 도구다. 에디터는 3컬럼 — 좌측 레일 / 가운데 다크 원고면 / 우측 작가진 검토 레일. 에디터 본체(상단 "일하는 바", 토큰, 원고면)는 이미 Editorial DS로 리디자인을 마쳤다. **이번 의뢰는 좌·우 레일 두 곳에 집중한다.**

## 직접 열어서 볼 것
- `https://sgeniusk.github.io/story-x-beta/?stage=editor` — 에디터. 좌측 레일, 우측 작가진 검토 레일, 작가진 카드 클릭 시 다이얼로그.
- `https://sgeniusk.github.io/story-x-beta/?stage=home` — 홈 플로우. 이 화면의 간결함이 톤 기준.
1280px·1440px 두 폭에서 확인한다. (LLM 생성·검토는 로컬 전용이라 공개 사이트에서는 동작하지 않는다. 레이아웃·시각만 본다.)

## 좌측 레일 — 회차 구성을 "구조적으로"

현재 — 작품 상태 카드 + "이번 회차 의도" + "회차 구성"(번호가 매겨진 평면 목록).

문제 (사용자가 직접 지적)
- 회차 목록을 좌측에 두는 건 의미가 없다. 회차 이동은 상단바에 있다. 좌측은 **편집의 구성·구간을 지칭하고, 그 단위로 작업**하게 하는 공간이어야 한다.
- 지금 "회차 구성"은 평면 번호 목록이라 구조가 안 보인다.

원하는 방향
- 회차의 내용을 **인지 가능한 구조 스킴으로 나눈다** — 기승전결, 또는 작품에 맞는 다른 구성. 각 구성 단위에 **AI가 정한 제목**이 붙는다.
- 작가는 그 구조의 흐름을 따라 **단위를 선택·이동**한다. 선택한 단위가 편집·도구의 기준이 된다.
- **위계와 시각을 정리** — 구조의 뼈대(상위 구성 → 하위 단위)가 한눈에 들어오게. 작가가 구조적으로 접근하도록 지원하는 것이 좌측 레일의 핵심 역할.
- 작품 상태·이번 회차 의도는 유지하되, **구성(structure)이 좌측 레일의 중심**이 된다.

## 우측 레일 — 글·정보에 집중, 가벼운 카드

현재 — 작가진 검토 카드: 픽셀 초상 + 이름 + 역할 + 단계 pill + 요약 + "대화하기".

문제 (사용자가 직접 지적)
- 검토 의견이 길어지면 카드가 한없이 길어지고 시각적으로 지저분하다.
- 카드가 너무 길다 (작가진 6명이 쌓이면 스크롤이 과해진다).

원하는 방향
- 카드의 박스 장식(chrome)을 줄이고 **글과 정보에 최대한 집중**한다. 클로드 디자인 자체 UI 같은 결 — 깔끔한 타이포 중심, 절제된 구분선, 불필요한 테두리·배경 최소화.
- 긴 검토 의견을 우아하게 — 카드가 무한정 길어지지 않게(요약 + 펼치기, 또는 절제된 타이포로 압축). 핵심을 먼저, 자세히는 펼쳐서.
- 작가진 6명이 쌓여도 한눈에 스캔 가능해야 한다.

## 유지할 것
- 3컬럼 + 작가진 사이드바 컨셉.
- 픽셀아트 작가진 캐릭터(브랜드 고정 결정).
- 실제 기능 — LLM 생성·검토, 작가진 1인씩 분리 검토, 작가진 다이얼로그, 캐논·바이블. 시각·구조만 바꾼다.
- 에디터의 Editorial DS 톤(warm cream 셸 + 다크 원고면 + 보라).

## 원하는 산출물
비평이 아니라 구체적 개선안 — 좌·우 레일 각각 무엇을/왜/어떻게. 색·간격·폰트·컴포넌트 구조까지 개발자가 옮길 수 있게. 클릭 가능한 HTML/CSS/JS 프로토타입.

## 제약
- 스택 Vite + React 18 + TypeScript, 단일 `src/styles.css`.
- 전면 리브랜딩이 아니라 적용된 Editorial DS 위에서 두 레일을 끌어올리는 작업.
