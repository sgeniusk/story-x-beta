# Story X — Notion DS 리디자인 체크리스트

출처는 클로드 디자인 핸드오프 번들 `StoryX Improved.html` (Notion 디자인 시스템). 라이트 랜딩·홈·목록 + 다크 에디터, P1~P5 개선이 핵심이다.

## 단계

- [x] 디자인 토큰 추가 — `:root`에 `--nx-*`(purple primary #6448d3, 캔버스 화이트, 파스텔 틴트, 8/12px 반경)
- [x] 랜딩 — 네이비 히어로 밴드 + 워크스페이스 목업 카드 + 파스텔 3카드 + 미디어 브릿지 + 클로징 (검증 완료)
- [x] 프로젝트 목록 — 흰 캔버스, 카드 그리드, 새 프로젝트 대시 카드 (검증 완료)
- [x] 홈 플로우 — 라이트 테마, 사이드바 노이즈 제거(P3), 단계 진행 인디케이터 (검증 완료)
- [x] 디자인 가드 테스트 갱신 — appExperience.test.ts, styles.montage.test.ts (126개 통과)
- [x] 에디터 P1 — 원고 영역 최상위(툴스트립 + 원고 전체), 의도는 좌측 레일 접이식 (검증 완료)
- [x] 에디터 P2 — 상단바 3컬럼, 우측 액션 2개(저장 칩 + 출간) (검증 완료)
- [x] 에디터 P5 — 트랙 전환 130ms 페이드, 집중 모드 버튼 상시 노출 (검증 완료)
- [x] 에디터 — 1080px 브레이크포인트(1280/1440에서 3열 유지), purple primary 액센트
- [x] 에디터 가드 테스트 갱신 — editorFocusLayout.test.ts (127개 통과)

## 남은 선택 사항

에디터 셸은 현재 라이트(다크 원고면 + 라이트 레일/상단바). 디자인 핸드오프의 에디터는 전면 다크(`#0b0e12`). 전면 다크 전환은 에디터 패널 전체를 다시 칠하는 큰 작업이라 회귀 위험이 있어 보류. P1~P5와 purple 액센트는 모두 반영됨. 사용자가 전면 다크를 원하면 별도 단계로.

## 원칙

- 시각·레이아웃만 디자인에서 가져온다. 실제 기능(LLM 인터뷰, building 단계, creativeWeight, 버전 히스토리, 포맷 선택)은 절대 후퇴시키지 않는다.
- 에디터 다크 토큰은 `.sx-desk` 스코프 유지. `:root` 라이트 토큰이 에디터를 깨면 안 된다.
- 한 화면씩 빌드 검증 후 다음으로.
