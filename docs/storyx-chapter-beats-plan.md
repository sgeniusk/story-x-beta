# Story X — 회차 구성(beat) 도입 계획

## 배경

에디터 좌측 레일의 편집 도구가 "이 단락"을 가리켰지만 "단락"의 정의가 없었다. 빈 줄로 갈리는 typographic 문단이 아니라, **이야기 흐름의 의미 단위**를 회차 생성 시점에 미리 잡고 번호를 매긴다. 그 번호 단위가 편집 도구·네비게이션의 기준이 된다.

## 결정

- 회차마다 **구성(beats)** — 번호 매겨진 흐름 단위 목록. 각 단위는 라벨 + 요약.
- 구성은 새 프로젝트 "구성" 단계에서 첫 초안과 함께 LLM이 생성한다. 이후 회차 생성도 동일.
- 원고와의 관계는 **오버레이** — 원고는 한 덩어리 textarea 유지. 구성은 좌측 레일의 번호 목록으로, 클릭하면 해당 대목으로 이동·하이라이트. 구조는 계획층, 글은 자유.
- 회차 목록은 상단바로 이관. 좌측 레일은 회차 구성 + 편집에 유용한 실제 도구.

## 단계

### 1단계 — 데이터 모델
- `storyEngine.ts` — `Chapter`에 `beats?: ChapterBeat[]`. `ChapterBeat = { id, no, label, summary }`.
- `DraftChapterPayload`·`chapterFromDraftPayload`에 beats 반영.
- `storyEngine.test.ts` 먼저 갱신(프로젝트 규칙).
- 시드 프로젝트·결정적 fallback에도 beats 채움.

### 2단계 — LLM 생성
- `tools/storyx.mjs` `buildDraftPrompt` — 초안 prose와 함께 회차 구성(beats) JSON을 반환.
- `draftClient.ts` — beats 파싱.

### 3단계 — 에디터
- 좌측 레일 — 회차 카드 목록 제거. 현재 회차의 **구성(번호 단위) 목록**으로 교체. 클릭 시 원고 해당 대목 이동·하이라이트. 작품 상태·캐논 건강도·이번 회차 의도는 유지.
- 상단바 — 브레드크럼에 회차 선택(드롭다운 또는 prev/next).
- 툴스트립의 중복 "흐름 검증" 액션 제거. 상단바 액션이 단일 검토 트리거. 검토 규모(Quick/Standard/Deep) 토글은 한 곳에만.
- 좌측 레일 도구는 실제 기능 있는 것만(분량 점검 등). 미보유 기능의 가짜 버튼 금지.

## 후속 — 구조 스킴 (2026-05-19 사용자 피드백)

평면 번호 목록을 넘어, beats를 **인지 가능한 구조 스킴**으로 키운다 — 기승전결 등으로 구성 단위를 묶고, 각 단위에 AI 제목을 붙인다. 좌측 레일은 그 구조의 위계를 보여주고 단위 선택의 기준이 된다. `ChapterBeat`에 act/구조 필드 추가, 초안 생성 프롬프트가 구조화된 아웃라인을 반환하도록 확장. 디자인은 `docs/storyx-editor-rails-brief.md`로 클로드 디자인에 의뢰.

## 원칙

- 가짜 UI 금지 — 기능 없는 버튼 안 만든다.
- 실제 기능(LLM 생성·검토, 버전 히스토리, 승인) 보존.
- 빌드·테스트 통과 유지.
