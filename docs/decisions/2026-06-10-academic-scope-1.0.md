## 옵션 비교

### 옵션 A — 1.0 에 full 포함

**의미.** A1~A5 를 정식 기능으로 출시하고, 라이브 검토 갭(미배선 페르소나)도 1.0 출시 전에 배선 완료. 학술 작품을 소설·에세이와 동등하게 완성 루프로 지원한다고 공개 약속.

**비용.**
- 라이브 검토 배선 — `04-media-breadth-smoke.md` 권고: "매체 특화 작가진을 라이브 검토에 배선 [중]". 논증 구조·인용 검토 페르소나 프롬프트 설계 + promptBuilders.ts/storyx.mjs 주입 + 테스트. 추산 medium effort.
- 학술 사용자 온보딩 검증 — academic 매체를 실사용하는 외부 작가(연구자·대학원생)가 소설 작가와 다름. 별도 온보딩 경로·UX 최적화가 필요.

**리스크.**
- `docs/reviews/2026-06-01-multiagent-review.md` 비즈니스 관점: "사회과학 학술 확장(A1~A5)이 연재 작가와 지불자가 다른 별도 제품 — 방향 분산" (medium severity). 소설/에세이 핵심 고객과 학술 사용자는 다른 사용 맥락·유료 의향 구조를 가진다.
- academic 트랙 전용 유지보수 부담(게이트 11종 + 추가 페르소나 + 학술 특화 온톨로지 확장)이 소설/에세이 개선에 쓰일 엔지니어링 예산을 감소시킨다.

**유지보수 부담.** 높음. academic 특화 모듈 4개(claimLedger·citationGate·academicIntegrity·academicPublish) + 추가 라이브 검토 페르소나가 독립적으로 유지·테스트되어야 한다.

### 옵션 B — 실험 기능 플래그로 포함

**의미.** A1~A5 를 `experimental` 또는 `beta` 플래그 아래 출시한다. 사용자는 설정에서 명시적으로 활성화해야 접근 가능. 공개 약속 범위를 "작동하나 미완성"으로 제한.

**비용.**
- 플래그 배선 — App.tsx 또는 projectBlueprint 에 feature flag 분기 추가. small~medium effort.
- 라이브 검토 갭은 플래그 아래 수용 가능 — 1.0 약속 범위에서 제외.

**리스크.**
- 플래그가 있으면 실제로 사용자가 발견·테스트하고 피드백을 줄 가능성이 낮아진다. "있지만 숨겨진" 기능은 외부 검증이 되지 않는다.
- A1~A5 구현이 이미 완결된 상태에서 플래그를 추가하는 것은 불필요한 복잡도 추가일 수 있다.

**유지보수 부담.** 중간. 플래그 분기 자체가 코드 경로를 둘로 나눠 회귀 위험을 높인다.

### 옵션 C — 1.0 제외, 1.1 이연

**의미.** 1.0 공개에서 academic 매체를 UI 상 숨기거나 비활성화. A1~A5 코드는 유지하되 사용자 접근 불가. 1.1 에서 라이브 검토 배선과 학술 사용자 검증을 완료한 후 공식 포함.

**비용.**
- 비활성화 — App.tsx 매체 선택 목록에서 academic 항목 조건부 제거. small effort.
- 코드 유지 — A1~A5 모듈은 테스트 포함 유지해야 한다(삭제 시 회귀 위험, 재구현 비용 발생). 유지 자체는 비용 없음.

**리스크.**
- 이미 완결된 A1~A5 를 1.0 에서 쓰지 않으면 구현 에너지가 낭비된다는 인상.
- 1.1 일정이 불확실하면 학술 기능이 장기 표류할 수 있다.

**유지보수 부담.** 낮음. 1.0 기간 동안 A1~A5 는 코드에 있지만 사용자 접근 없음 → 실제 유지보수 압력 최소.

## 권고안 — 옵션 B (실험 플래그 포함) with 조건

### 권고

**1.0 에 실험 플래그(beta) 로 포함한다.** 단, 다음 두 조건이 1.0 출시 전에 충족될 경우에만.

1. **라이브 검토 갭(미배선 페르소나) 배선 완료** — `04-media-breadth-smoke.md` 권고 [중] 이행. 학술 본문을 소설 5인이 검토하는 상태 해소.
2. **온보딩에 "실험 기능" 명시** — 사용자가 academic 을 활성화할 때 "아직 검증 중인 기능입니다" 텍스트 표시.

조건 충족이 1.0 일정 내 어렵다면 옵션 C(1.1 이연)로 후퇴한다.

### 근거

**소설·에세이 핵심 루프 우선.** `docs/reviews/2026-06-01-multiagent-review.md` 비즈니스 관점은 "사회과학 학술 확장이 연재 작가와 지불자가 다른 별도 제품 — 방향 분산"을 medium severity 로 기록했다. 1.0 의 시장증명 대상은 소설/에세이 연재 작가다. academic 이 여기에 분산을 일으키면 rank6 done_criteria 검증이 어려워진다.

**실험 플래그가 옵션 A 대비 낫다.** 라이브 검토 갭이 1.0 에서 사용자에게 노출되면 "게이트엔 있지만 검토엔 없다"는 주장과 실제의 괴리가 재발한다. 이는 `docs/reviews/2026-06-01-multiagent-review.md` 총괄 진단이 "가장 치명적"으로 분류한 패턴이다. 플래그로 약속 범위를 제한하면 이 위험을 완화한다.

**이미 완결된 A1~A5 를 숨기는 비용은 낮다.** 코드 삭제 없이 UI 접근만 조건부로 제한하면 되므로 구현 에너지 낭비는 최소다. 1.1 에서 라이브 검토 배선 + 외부 학술 사용자 검증 후 정식 포함이 가능하다.

**사용자 수요 검증 비용.** 학술 사용자(연구자·대학원생·논문 작가)는 소설 작가와 다른 채널에 있다. 1.0 소설/에세이 시장증명 외에 학술 수요를 별도 검증하는 것은 1.0 일정에 과부하다. 실험 플래그로 1.0 에 두고 피드백을 수집하는 것이 비용 대비 효율적이다.
