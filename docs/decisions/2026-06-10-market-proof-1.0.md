# M7-alpha-1.0 done_criteria 재정의 결정안

> 작성 2026-06-10 · 근거 문서 `docs/reviews/2026-06-01-multiagent-review.md` rank6 · `docs/reviews/2026-06-07-persona-live-test/FINAL-REPORT-romancefantasy.md` · `docs/reviews/2026-06-07-persona-live-test/04-media-breadth-smoke.md`

## 현행 기준의 문제

M7-alpha-1.0 의 현행 `done_criteria` 는 다음이다.

> "30화 시리즈 회귀 통과, 완성 루프 데모 가능" (`feature_list.json` id: M7-alpha-1.0)

이 기준은 두 가지 이유에서 시장 증명으로 불충분하다.

**1. 기술 게이트만으로는 "사용자가 작품을 완성할 수 있는가"를 측정하지 못한다.**

`docs/reviews/2026-06-01-multiagent-review.md` 비즈니스 관점(rank6) 인용.

> "1.0 완료 기준이 1인 사용 만족 + 기술 회귀 — 시장 신호 전무" (severity high · effort medium)
> "런칭 플랜 1단계 완료 기준이 사실상 1인(제작자) 만족이고 과금·외부 검증을 전부 2단계로 미룬다."

회귀 30화가 통과해도 외부 작가가 첫 세션에서 이탈하거나, 완결까지 도달하지 못하면 제품은 시장에서 검증된 것이 아니다.

**2. "완성 루프 데모 가능"은 제작자 내부 시연이지 독립 검증이 아니다.**

같은 문서 총괄 진단 인용.

> "주장과 실제의 괴리 — 게이트가 하드코딩 리터럴을 평가한다. … 제품의 유일한 차별점 '연속성을 제품 요건으로'가 데모에선 작동하는 듯 보이나 실제 판정력이 비어 있다."

데모 가능성이 아니라 실제 작가가 작품을 완성했다는 증거가 필요하다.

## 내부 실증이 이미 달성한 것

페르소나 실증 테스트(2026-06-07~08)에서 내부 기준 완권은 이미 달성됐다.

- `FINAL-REPORT-romancefantasy.md` 한 줄 결론: "Story X의 차별점('연속성=제품 요건')은 **21화에 걸쳐 결정적으로 입증됐다**."
- 6축 점수 연속성 5 / 평균 ~3.8 · **23화 완결**(전제 이행·수미상관·연속성 무결점).
- `04-media-breadth-smoke.md` 결론: "**차별점(연속성/캐논)은 5매체 공통 코어로 작동**", 매체 전수(소설·만화·에세이·오디오·학술) 생성·인물추출·매체적합성 확인.

이 결과는 "내부 시연 가능" 수준을 넘어 **내부 실증 완료**로 분류할 수 있다. 다음 단계는 외부 실증이다.

## 재정의안 — 2단계 시장증명 게이트

### 단계 1 — 기술 게이트 (현행 유지)
- 30화 시리즈 회귀 전체 통과
- `npm test` 전체 녹색 · `npm run build` 성공
- 완성 루프(인터뷰→생성→검토→퍼블리시) 각 매체 1작품 이상 end-to-end 검증
- 내부 페르소나 실증에서 연속성 5점 기준 작품 1편 이상 달성 (이미 달성: #2 로판 23화 완권)

### 단계 2 — 외부 실증 게이트 (신규 추가)
- **외부 작가 3명** 이상이 Story X 를 사용해 **각 1편(10화 이상) 완성** — 제작자 개입 없이 독립 완주
- 완성 기준: 인터뷰→초안→검토→의도 메모 활용이 포함된 실 작업 흔적(로그·스냅샷) 존재
- 매체 제한 없음(소설·에세이·만화·오디오·학술 모두 인정)
- 완성 작품 중 **1편 이상이 소설 또는 에세이** (핵심 완성 루프 커버)
- 3명 중 **1명 이상이 유료 지속 의향 표명** 또는 실 과금 이력 보유 — 인터뷰·설문·과금 기록 중 하나

**N 수치 근거:** 검토 문서 rank6 권고 "외부 작가 3~5명 핸즈온 검증" 하한을 채택. 최소 단계에서 신호를 얻고 상향 조정 가능.

## 경량 검증 방법 — 1.0 전에 비용 낮게 시장 신호 얻기

1.0 공식 릴리스 전에 외부 실증을 준비하는 세 가지 구체 방법과 성공 지표.

### 방법 A — 페르소나 실증 로그 선별 공개
**방식.** `FINAL-REPORT-romancefantasy.md` 와 `04-media-breadth-smoke.md` 의 대화·로그 일부를 GitHub 또는 블로그에 공개한다. 연속성·캐논 메커니즘이 실제로 작동하는 증거를 외부에 노출한다.
**성공 지표.** 공개 후 2주 내 외부 작가 1명 이상이 접촉(이메일·DM·이슈) — 관심 있는 베타 작가 파이프라인의 최소 신호.
**비용.** 로그 편집·마크다운 정리 ~2~4시간. 인프라 추가 없음.

### 방법 B — 소수 베타 작가 모집(클로즈드 베타 3~5명)
**방식.** 글쓰기 커뮤니티(문피아 작가 포럼·네이버 시리즈 작가 카페 등) 또는 지인 네트워크에서 **연재 경험 있는 작가 3~5명**을 직접 모집한다. BYO 키(BYOK) 방식으로 LLM 원가를 작가 부담으로 진행 — 서버 인프라 투자 전에 검증.
**성공 지표.** 3명이 2주 내 10화 이상 완성 시도, 6축 평균 3.5 이상. 이탈 이유가 기록됨(버그·UX·생성품질 분류).
**비용.** 모집 연락 ~2~4시간 + 온보딩 지원 ~2시간/인. BYOK 이면 LLM 원가 0.
**근거.** `docs/reviews/2026-06-01-multiagent-review.md` rank6: "서버 인프라 착수 전 외부 작가 3~5명 핸즈온 검증을 끼워라."

### 방법 C — 완성 루프 데모 영상(5분 이내)
**방식.** 내부 페르소나 실증(#2 로판 23화 완권)에서 가장 인상적인 장면—캐논 고정, 연속성 감수자 충돌 포착, 완결 reveal—을 Loom 또는 YouTube 비공개 영상으로 녹화한다. 외부 작가 모집 시 첫 소개 자료로 활용하고, 피드백으로 작가 기대치와 제품 실제 동작의 차이를 측정한다.
**성공 지표.** 영상 시청 후 "써보고 싶다"는 반응 비율 ≥50% (조회한 5명 중 3명 이상) — 베타 참가 의향과 직결.
**비용.** 녹화·편집 ~2~3시간. 인프라 없음.

## 미확인 사항

- 외부 베타 작가 모집 채널·시기가 결정되지 않았다 — 사용자 결정 필요.
- 유료 지속 의향의 구체 과금 구조(월정액·토큰 단위 등)가 feature_list 또는 launch-plan 에 없다 — 확정 전까지 "지속 의향 표명"을 게이트로 유지.
- rank6 검토는 "과금 가설 부재 — 누가 얼마를 왜가 코드·문서에 없음"을 high severity 로 기록했다(출처: `docs/reviews/2026-06-01-multiagent-review.md` 비즈니스 관점). 과금 구조가 정의되면 유료 지속 의향 게이트도 구체화 필요.
