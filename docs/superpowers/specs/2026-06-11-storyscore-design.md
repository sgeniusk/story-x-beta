# StoryScore v0.1 — 작품 완성도 공식 점수 시스템 설계

> 2026-06-11 사용자 결정 — "AI 시스템 평가하듯 작품 완성도를 평가하는 시스템을 만들고, 스킬화해서 공식 점수화하자."
> 첫 적용 대상 — 쇼케이스 30화 A/B: 실험군(Story X+codex "철거 전야의 이름") vs 통제군(맨 Claude 직접 집필, `/tmp/storyx-control-claude/`).

## 원칙

1. **입력 중립** — Story X 백업 JSON(두 형식)과 맨 마크다운 디렉토리(`chNN.md`)를 같은 내부 표현(`{episode, title, prose}[]`)으로 정규화해 채점한다. 하네스 산출 메타데이터(canonFacts·stakesLedger)는 **공식 점수에 쓰지 않는다**(입력 비대칭 — 통제군에 없음). 부가 진단으로만 출력.
2. **결정론 + LLM 심사 분리** — 코드로 잴 수 있는 것은 코드로(재현 가능), 의미 판단은 루브릭 기반 LLM 심사로. 심사는 근거 인용(회차·문장)이 의무다.
3. **버전 고정** — 점수 공식·루브릭은 `STORYSCORE_V0_1` 로 박제. 공식이 바뀌면 버전을 올리고 과거 점수와 섞지 않는다.

## 점수 공식 (100점)

| 축 | 배점 | 측정 방식 |
|---|---|---|
| 연속성 | 30 | 심사(루브릭 1~5 → 환산) + 결정론 보조(로스터 출현 매트릭스·철자 변형 의심 감점) |
| 페이오프/구조 | 25 | 심사 — 떡밥 대차(발급/회수 목록 추출→회수율), 결말의 전제 이행 |
| 프로즈 품질 | 20 | 심사 — 표본 회차 루브릭(문체 일관·장면 구체성·대사·과설명) |
| 연재성/후크 | 15 | 결정론 후크 휴리스틱(7) + 심사(8) |
| 형식 규율 | 10 | 결정론 — 회차 분량 분포(목표 1,800~2,700자 적합률)·형식 준수 |

- 심사 루브릭 각 1~5점, 근거 인용 없으면 무효(재심사).
- 표본 — 전 회차 결정론 스캔 + 심사는 5표본(1화·25%·50%·75%·최종화) 깊이 읽기 + 전체 줄거리 연속(제목·말미 시퀀스)훑기.

## 구성물

1. **CLI `tools/storyscore.mjs`** (TDD — `src/storyscoreCli.test.ts` 또는 tools 테스트 컨벤션 따름)
   - `--input <경로>` — 백업 JSON(키맵/dump)·md 디렉토리 자동 감지 정규화
   - `--roster "이름1,이름2"` (선택, 미지정 시 입력 메타에서 추출 가능하면 추출)
   - 결정론 산출 — 분량 분포·적합률, 로스터×회차 출현 매트릭스, 철자 변형 의심(접두 일치+꼬리 1자 차이), 말미 후크 휴리스틱(질문/말줄임/전환 신호), (있으면) 하네스 메타 부가 진단
   - `--out report.json` — 결정론 부분 점수 + 심사용 표본 패키지(표본 회차 본문·전회차 제목/말미 200자)를 한 JSON 으로
2. **스킬 `.claude/skills/story-score/SKILL.md`** — 공식 채점 절차의 정본
   - 절차: CLI 실행 → report.json 의 결정론 점수 채택 → 표본 패키지를 루브릭으로 심사(각 축 1~5, 근거 인용 의무) → 공식 합산 → 스코어카드 마크다운 출력
   - 루브릭 전문(각 축의 1·3·5점 기준 서술)을 스킬 안에 포함 — 심사자가 바뀌어도 잣대 고정
   - 출력 형식: `STORYSCORE_V0_1 — 총점 NN/100` + 축별 표 + 근거 인용 + 결정론 진단 부록

## 공정성 메모 (A/B 적용 시)

- 양쪽 같은 표본 규칙·같은 루브릭·같은 심사 모델로 1회씩. 심사자는 어느 쪽이 실험군인지 모르게 입력 라벨을 중립화(`작품 A`/`작품 B`).
- 통제군은 단일 컨텍스트 집필이라 초반 유리할 수 있음 — 연속성 축은 후반(75%·최종) 표본 가중이 중요하다는 점을 루브릭에 명시.
