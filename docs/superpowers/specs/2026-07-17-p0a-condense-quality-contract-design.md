# P0-a 후속 — PLAY 응결 장면화·보이스 품질 계약

> Date: 2026-07-17 · Status: accepted from live dogfooding · Scope: prompt/context first slice

## 문제

실제 응결본 「옥상에 가지 않는 첫날」은 1화 약속은 선명했지만 1,122자, 직접 대사 0, 사건 보고와 추상 해설 중심이었다. `P0B-ALPHA/BETA` 검증 표식도 작품 속 증거와 캐논 후보로 승격됐다. 이는 모델 편차보다 현재 지시의 결과다.

- `3인칭 서사 회차로 압축`, `대사를 그대로 옮기지 말라`, `즉흥 설정도 버리지 말고 봉합`은 요약·합리화에 강한 보상을 준다.
- 장면 수·대화·압력 상승·선택의 대가·구체 후크·목표 분량 계약이 없다.
- `buildProjectContextDigest`는 `koreanVoiceRules`를 계산하지만 출력하지 않아 사용자의 tone/rhythm/vocab가 응결 프롬프트에 도달하지 않는다.
- 최신 두 메시지 제외와 승인 전 품질 게이트도 문제지만, 소비 경계와 오탐 정책을 바꾸므로 이 첫 슬라이스와 분리한다.

## 설계 질문과 결정

### 1. 이번 수정이 원고를 자동으로 거부하거나 재생성할 것인가?

아니다. 첫 슬라이스는 입력 계약과 프롬프트를 강화한다. 분량·대화·후크 휴리스틱은 이후 경고형 readiness로 별도 도입하며, 오탐이 검증되기 전 캐논 승인을 새 규칙으로 막지 않는다.

### 2. `압축`은 무엇을 뜻하는가?

PLAY의 반복·머뭇거림을 덜어내되 사건을 요약문으로 줄이는 뜻이 아니다. 일어난 사실·감정 변화·약속을 2~3개의 현재 장면으로 다시 배열하고, 행동·감각·인물별 대사로 장면화한다.

### 3. 장편 소설 1화의 기본 산출 계약은 무엇인가?

- 한국어 본문 최종 허용 범위는 1,800~2,700자이며, 모델 계산 편차를 감안한 실제 생성 목표는 1,900~2,600자로 둔다. 재료가 부족할 때 설정을 발명해 채우지 말고 짧더라도 사실 범위를 지킨다.
- 완료 전에 prose 글자 수를 확인한다. 1,800자 미만이면 새 설정·기능 해설·반복으로 채우지 않고, 입력에 이미 있는 장면의 행동·감각·갈등 대사를 구체화한다. 입력 재료가 실제로 부족할 때만 짧은 결과를 허용한다.
- 서로 다른 압력을 가진 현재 장면 2~3개.
- 원문에 인물 대화가 있으면 갈등을 가진 직접 대화 교환 최소 1회. 원문에 없는 핵심 사실을 대사로 발명하지 않는다.
- 각 장면은 압력 상승·선택 축소·실제 대가 중 하나를 만든다. 같은 장면 안에서 모두가 합리적으로 합의해 갈등을 봉합하지 않는다.
- 한 질문을 답하고 더 날카로운 질문을 연다. 말미 2~3문장은 구체적인 역전·위협·시한·불가피한 선택 중 하나이며 주제 요약으로 닫지 않는다.

### 4. 메타 표식은 어떻게 막는가?

임의 정규식으로 `ALPHA` 같은 단어를 삭제하지 않는다. 입력이 테스트·QA·복구·UI 안내·스키마·타임스탬프라고 스스로 표시한 텍스트는, 사용자가 작품 내부 고유명사라고 명시 승인하지 않는 한 title/hook/prose/canon에서 제외하도록 프롬프트 계약을 둔다. 구조적 turn provenance는 후속 슬라이스다.

### 5. 캐논 발명은 어디까지 허용하는가?

`newCanonFacts`에는 원문·현재 장면·기존 캐논에 명시된 사실과 장면 성립에 꼭 필요한 최소 추론만 둔다. 새 이름·기관·범행 원인·시간·관계는 근거가 없으면 본문 편의로도 확정하지 않고 캐논 후보에 넣지 않는다.

### 6. 사용자 목소리는 어떻게 반영하는가?

바이블의 tone/rhythm/vocab 섹션을 `한국어 문체·보이스 규칙`으로 실제 context digest에 출력한다. 응결은 이를 마지막 문장 패스의 우선 규칙으로 사용한다. 인물 카드에는 욕망·상처·현재 상태·말투·캐논 앵커를 함께 전달한다.

첫 실생성의 독립 보이스 감수에서 구조는 통과했지만, 물리 묘사 뒤 감정명 재설명·근거 없는 높임말 전환·선택지의 기능적 재요약이 P1로 발견됐다. 따라서 마지막 패스는 단순한 "문체 반영"이 아니라 다음 검수 계약을 함께 가진다.

- 물리 신호로 이미 드러낸 감정을 감정 이름으로 다시 설명하지 않고, 바이블 금지어를 최종 원고에서 제거한다.
- 인물별 호칭·높임말·말끝은 원문과 캐릭터 카드의 근거를 유지한다.
- 장면 연결과 말미의 질문·선택은 기능적으로 해설하지 않고 사물·행동·대사로 보여 준다. 인과 설명 뒤 결정은 구체 동사의 독립 단문으로 둔다.
- 캐릭터 카드의 욕망·말투·우선순위를 서술자가 이행 보고처럼 다시 설명하지 않는다. 장면의 `질문`·`문제`·`선택`·`답`을 prose에서 기능어로 명명하지 않고, 드러난 딜레마 뒤에는 비교 해설 없이 행동한다.
- 말투 규칙은 실제 대사 배열에만 적용하며, 대사 앞뒤에서 문장 길이·말끝·호칭 변화를 서술자가 해설하지 않는다.
- 마지막 2~3문장은 각각 하나의 구체 행동·감각·시한을 담는 독립문으로 두고, 행동 직전 의미·동기·선택을 해설하지 않는다.

## 수용 기준

- `buildProjectContextDigest`가 저장된 tone/rhythm/vocab 규칙을 실제 문자열로 출력한다.
- 로컬 CLI와 TypeScript prompt builder가 핵심 응결 계약을 byte-identical 문장으로 미러한다.
- 프롬프트가 1,800~2,700자, 2~3 장면, 직접 대화, 압력/대가, 구체 후크, 메타 제외, 캐논 근거 제한, 보이스 적용을 모두 포함한다.
- 보이스 최종 검수가 감정 재설명·금지어 잔존·호칭/높임말 흔들림·선택의 기능적 요약을 명시적으로 막는다.
- 캐릭터 카드 이행 보고와 딜레마 재해설을 막고, 말미 2~3문장을 독립된 구체 사건으로 제한한다.
- 캐릭터 입력에 wound/currentState/canonAnchors가 포함된다.
- 기존 JSON 출력 계약과 캐논 검토·사용자 승인 게이트는 바뀌지 않는다.
- 집중 테스트, 실제 로컬 Codex 재응결, 전체 `bash init.sh`가 녹색이다.

## 후속 분리

- 최신 2턴을 응결 본문과 다음 PLAY 연결에 중복 없이 사용하는 소비 경계.
- 분량·직접 대화·추상 말미·메타 누수의 승인 전 경고/readiness.
- sourceTurnIds와 explicit/necessary-inference/invention provenance를 가진 캐논 후보 스키마.
