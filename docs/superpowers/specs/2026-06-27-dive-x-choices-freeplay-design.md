# Dive X 진행 선택지 + 흘려보내기 + 자유 응결 (묶음 A) 설계

> 자연어 입력을 기본으로 유지하되 타이핑 부담을 덜고, 사용자가 자유롭게 들이민 비현실·즉흥 입력을 응결 때 매끄럽게 봉합한다. dogfooding 중 사용자 통찰에서 출발.

## 1. 핵심 원칙 (확정)

- **자연어가 항상 기본.** 선택지 칩·계속 버튼은 "내가 내 말과 행동을 자연어로 고른다"는 본질 위의 *보조*일 뿐. 타이핑은 늘 가능하고, 칩은 무시해도 된다.
- **자유 응결이 도파민 코어.** 사용자가 막 들이민 비현실·즉흥 입력(예 — "근처 총을 들고 쐈다")도 막지 않는다. 응결(고급 모델 단계)이 그걸 **기존 캐논과 모순되지 않는 선에서 그럴듯하게 봉합**해 매끄러운 소설로 만든다. 자유롭게 구부리고 이야기가 우아하게 흡수하는 순간이 재미다.
- **연속성은 죽이지 않는다(긴장의 원천).** 하드 캐논(예 — *방탄피부*)은 그대로 살아 마찰을 만든다. 그 마찰을 영리한 플레이로 풀거나, **프리미엄(묶음 B)**으로 캐논을 갈아엎거나 과거로 되돌린다. 본 스펙의 자유 응결은 캐논을 *지우지 않고* 자유 입력을 캐논 안에서 봉합한다.

## 2. 추가하는 것

### 2.1 ⏳ 계속 / 흘려보내기 버튼 (항상)
- 항상 노출되는 버튼. 누르면 암묵 행동 `(가만히 지켜본다. 시간이 잠시 흐른다.)`를 사용자 메시지로 전송 → 쇼러너가 세계·인물을 한 박자 진전.
- 내부적으로 기존 `send(text)` 재사용(특수 분기 없음).

### 2.2 행동 선택지 칩 (맥락형 · 인라인)
- `dive-chat` 출력을 `{ reply, choices: string[] }`로 확장. 모델이 응답과 함께 **주인공("나")이 이 장면에서 자연스럽게 취할 만한 행동·말 2~3개**를 짧은 동사구로 제안. **추가 LLM 호출 없음**(같은 응답에 묻어옴).
- 최신 캐릭터 턴 아래 탭 가능한 칩으로 렌더. **탭 = `send(칩텍스트)`**(평범한 한 턴). 새 메시지를 보내거나 응결/쇼러너 동작 시 칩은 사라진다.
- 칩은 보조 — 무시하고 자유 입력 가능. busy 중엔 비활성.

### 2.3 자유 응결 (dive-condense 프롬프트 강화)
- dive-condense 프롬프트에 한 줄 추가 — "사용자가 자유롭게 들이민 비현실·즉흥 설정도 버리지 말고, **기존 캐논(현재 장면·확정 사실)과 모순되지 않는 선에서 그럴듯한 인과로 봉합**해 매끄러운 3인칭 소설로 재구성하라. 단 확정된 하드 캐논을 임의로 뒤집지는 말 것."
- 하드 캐논 충돌은 봉합 대신 긴장으로 남는다(해소는 묶음 B 프리미엄).

## 3. 시스템 설계 — 작은 단위

### 3.1 수정 단위 (전부 가벼움)
- **`tools/storyx.mjs`**
  - `dive-chat` — 프롬프트에 choices 지시 추가, 출력 객체에 `choices: normalizeStringList(parsed?.choices)`. mock은 샘플 choices 2~3개.
  - `dive-condense` — §2.3 자유 응결 한 줄 추가.
- **`src/lib/diveClient.ts`** — `DiveChatResponse`에 `choices?: string[]` 추가.
- **`src/components/DiveDesk.tsx`**
  - 상태 `choices: string[]` 추가. `send()` 시작 시 `setChoices([])`, dive-chat 응답 후 `setChoices(res.choices ?? [])`.
  - 최신 캐릭터 턴 아래(채팅과 컴포저 사이) 칩 바 — `choices.length>0 && !busy` 일 때. 칩 onClick → `send(choice)`.
  - 컴포저에 `⏳ 계속` 버튼 → `send('(가만히 지켜본다. 시간이 잠시 흐른다.)')`. busy/pending 중 비활성.
- **`src/styles.css`** — `.dx-choices`(칩 바) · `.dx-choice-chip` · `.dx-continue`.

### 3.2 데이터 흐름
사용자 입력(타이핑/칩탭/계속) → `send(text)` → `requestDiveChat({scene, ...})` → `{reply, choices}` → reply는 세그먼트 렌더, choices는 칩 바 갱신. 자유 입력의 비현실 설정은 그대로 누적되다 응결 때 봉합.

## 4. 비목표 (YAGNI · 묶음 B 또는 이후)
되돌리기(과거 복원) · 쇼러너 캐논 god-편집(설정 삭제/변경) · 실결제·포인트 차감 · 분기 트리·선택지 결과 미리보기 · 멀티 선택 · 고정 프리셋 칩 · 칩을 캐릭터별로 분리.

## 5. 리스크
- **choices 품질** — 모델이 맥락 약한/중복 선택지를 낼 수 있음. 짧은 동사구 지시 + 무시 가능(자연어 기본)이라 치명적 아님.
- **자유 응결 과봉합** — 너무 매끄럽게 봉합하면 긴장이 사라질 수 있음. "하드 캐논은 뒤집지 말 것" 가드로 균형. 실제 균형은 dogfooding으로 조정.
- **칩 타이밍** — busy 중·새 입력 시 사라지는 규칙이 어긋나면 헷갈림. 단순 규칙(보낼 때 clear, 응답 때 set)로 고정.
