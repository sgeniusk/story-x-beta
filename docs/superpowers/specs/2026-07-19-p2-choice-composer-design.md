# P2-c1 선택지 칩 → 작성창 설계

> 날짜: 2026-07-19
> 상태: brainstorm 결정 완료
> 브랜치: `codex/p2-choice-composer`
> 기준: Draft PR #44 (`codex/p1b-text-export`) 위 단일 슬라이스

## 문제

PLAY 응답의 추천 선택지는 현재 클릭 즉시 사용자 턴으로 저장되고 다음 LLM 요청을 시작한다. 사용자는 추천 문장을 자기 말투로 다듬을 수 없고, 응답 도착 때의 레이아웃 변화로 잘못 누르면 약 30~40초짜리 요청 한 번을 되돌릴 수도 없다. 이는 Story X가 약속한 창작 주도권과 보이스 보존을 직접 훼손한다.

이번 슬라이스는 추천 선택지를 전송 버튼이 아니라 **수정 가능한 답변 재료**로 바꾼다. 생성 프롬프트·대화 응답·캐논·응결 계약은 바꾸지 않는다.

## Brainstorm 결정

### 질문 1 — 어떤 선택지를 바꾸는가?

**결정:** `dive-chat` 응답의 가벼운 `choices` 칩만 바꾼다. `⏳ 계속`, `⏭ 전개`, VS 전개 후보는 명시적 장면 진행 명령이므로 기존 즉시 실행 의미를 유지한다.

### 질문 2 — 칩을 누르면 무엇이 일어나는가?

**결정:** 선택 문구를 PLAY 작성창에 정확히 담고 작성창으로 포커스를 옮긴다. 이 단계에서는 fetch, `onChange`, 대화 턴 추가, 진행 작업등을 전혀 시작하지 않는다. 사용자가 문구를 고친 뒤 Enter 또는 `보내기`를 눌러야만 기존 `send()` 계약이 실행된다.

### 질문 3 — 사용자가 이미 답변을 쓰고 있으면 어떻게 하는가?

**결정:** 비어 있지 않은 작성창이 있으면 추천 칩을 숨겨 기존 문장을 byte-for-byte 보존한다. 자동 덮어쓰기나 자동 이어붙이기는 하지 않는다. 사용자가 작성창을 다시 비우면 아직 전송하지 않은 추천 칩을 다시 보여 준다.

### 질문 4 — 선택한 뒤 다른 추천으로 돌아갈 수 있는가?

**결정:** 칩을 눌러도 `choices` 상태는 전송 전까지 보존한다. 선택 문구가 작성창에 있어 칩은 잠시 숨지만, 사용자가 문구를 모두 지우면 같은 추천을 다시 고를 수 있다. 명시 전송할 때는 기존 `send()`가 추천 목록을 정리한다.

### 질문 5 — 사용자가 새 동작을 어떻게 알아채는가?

**결정:** 칩 위에 `추천 답변 · 눌러 작성창에 담아 고치세요`라는 짧은 안내를 둔다. 칩은 버튼이고 작성창 이동 의미를 title/접근성 이름으로도 제공한다. 선택 직후 작성창 포커스와 커서 끝 위치로 바로 수정할 수 있게 한다.

### 질문 6 — 실패와 동시성은 어떻게 다루는가?

**결정:** 칩 선택 자체에는 실패 가능한 외부 작업이 없다. busy·응결 검토 중에는 기존처럼 칩을 노출하지 않는다. 사용자가 수정본을 전송한 뒤 네트워크가 실패하면 기존 대화 실패 계약을 그대로 따르며, 이번 슬라이스에서 요청 재시도·입력 복원 정책은 바꾸지 않는다.

## UI 계약

- 추천 영역은 `choices.length > 0`, `!busy`, `pending === null`, `input.trim() === ''`일 때만 렌더한다.
- 칩 클릭은 `input`만 선택 문구로 바꾸고 PLAY 작성창을 focus한다.
- 작성창 값이 공백뿐이면 추천을 담을 수 있는 빈 상태로 본다.
- 칩 선택만으로 사용자 버블, 네트워크 요청, 진행 카드가 생기지 않는다.
- 작성창은 기존 Enter 전송·Shift+Enter 줄바꿈 계약을 유지한다.
- 한글 IME 조합 중 Enter는 글자 확정만 수행하고 답변을 전송하지 않는다.
- 안내와 칩은 warm studio `--st-*` 토큰 및 기존 PLAY 선택지 시각 언어를 재사용한다.

## 수용 기준

- 실제 응답에서 추천 칩이 보이고 행동 의미 안내가 함께 보인다.
- 칩 클릭 뒤 fetch·`onChange` 호출 수는 증가하지 않고 textarea 값과 focus만 바뀐다.
- 추천 문구를 고친 뒤 `보내기` 또는 Enter를 누르면 수정본이 단 한 번 전송된다.
- 한글 조합을 확정하는 Enter에서는 네트워크·대화 변경이 없고 입력이 보존된다.
- 사용자가 먼저 입력하면 추천 칩이 숨고 기존 문장이 보존되며, 모두 지우면 추천이 다시 나타난다.
- `⏳ 계속`, `⏭ 전개`, VS 후보의 기존 실행 의미는 회귀하지 않는다.
- 1280·390·320px에서 안내·칩·sticky 작성창이 겹치거나 가로로 넘치지 않는다.
- focused 테스트, `bash init.sh`, 독립 코드/브라우저 감사가 녹색이다.
