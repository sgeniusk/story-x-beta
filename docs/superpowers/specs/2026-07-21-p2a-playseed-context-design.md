# P2-a PLAY 시드 확인 카드 맥락 정리 설계

> 날짜: 2026-07-21
> 상태: brainstorm 결정 완료
> 브랜치: `codex/p2a-playseed-provenance`
> 기준: Draft PR #47 (`codex/sites-private-pilot`) 위 단일 슬라이스

## 문제

인기 프리셋이나 함께 구상에서 이미 `DiveSetup`을 확정한 뒤 PLAY 시드 확인 카드로 들어와도, 화면 상단에 초기 Dive X 범용 상대인 도윤·하란·세하가 같은 위계로 항상 나타난다. 이 칩은 현재 설정과 무관할 뿐 아니라 클릭하면 첫 장면·내 역할·인물 전체를 별도 세계관으로 조용히 교체한다.

2026-07-13 실플레이에서는 악역 영애 프리셋과 함께 구상 두 경로에서 재현됐고, 5개 페르소나 모두 세계관 몰입과 제품 신뢰를 깎는 결함으로 지적했다. 이번 슬라이스는 확인 단계를 **현재 선택한 설정을 검토하고 대화 상대를 고르는 곳**으로 되돌린다.

## Brainstorm 결정

### 질문 1 — 확인 카드에서 범용 상대 3종을 언제 보여 주는가?

**결정:** 보여 주지 않는다. 확인 단계에는 현재 `setup.scene`, `setup.myRole`, `setup.cast`만 노출한다. 도윤·하란·세하는 현재 작품의 후보가 아니며 같은 화면에 섞지 않는다.

### 질문 2 — setup이 없거나 생성에 실패했을 때 범용 칩을 구제 경로로 남길 것인가?

**결정:** 남기지 않는다. 사용자는 `이전`으로 소재발굴 단계에 돌아가 인기 프리셋 5종 또는 함께 구상을 다시 선택한다. 확인 단계 안에서 출처가 다른 세계관으로 교체하는 두 번째 선택기를 두지 않는다. `setup`이 없으면 기존처럼 `이대로 시작`을 비활성화하고 오류·대기 안내만 정직하게 보여 준다.

### 질문 3 — `DIVE_SEED_CHARACTERS`와 `presetToDiveSetup` 자체도 삭제하는가?

**결정:** 삭제하지 않는다. 기존 Dive 세션 복원과 도메인 호환 테스트가 사용하는 자산이다. 이번에는 `App → PlaySeedPanel`의 확인 카드 배선만 제거한다.

### 질문 4 — 현재 설정 안의 상대 선택은 어떻게 되는가?

**결정:** `setup.cast` 버튼, `partnerIndex`, `aria-pressed`, 선택 후 `buildPlayFirstProject(..., partnerIndex)` 의미를 그대로 유지한다. 확인 카드의 핵심 행동은 바꾸지 않는다.

### 질문 5 — 무엇을 회귀 기준으로 삼는가?

**결정:** 특정 이름이 없다는 단언만 쓰지 않는다. 실제 작품 인물 이름이 우연히 도윤일 수 있으므로, 범용 그룹 클래스·접근성 이름·관련 props가 사라지고 contextual cast 버튼 수가 `setup.cast.length`와 같다는 구조를 테스트한다.

### 질문 6 — 인접한 P2 결함도 함께 고치는가?

**결정:** 고치지 않는다. 건너뛴 인터뷰의 완료 표시, 프로젝트 제목 절단, PLAY 추천 답변, 내부 용어·죄책감 UI, 상대 추가·커스텀, S3 적응형 인터뷰는 각각 별도 슬라이스다.

## UI·데이터 계약

- PLAY 시드 확인 화면에는 현재 `DiveSetup`에서 파생한 첫 장면·내 역할·대화 상대만 보인다.
- 확인 화면은 `DIVE_SEED_CHARACTERS` 또는 `presetToDiveSetup`을 받거나 호출하지 않는다.
- `setup === null`이면 확인 버튼은 비활성화되고 오류·loading 안내 계약은 유지된다.
- `setup.cast`가 여러 명이면 기존 선택 버튼과 선택 상태를 그대로 제공한다.
- 인기 프리셋과 함께 구상은 각각 확정한 setup을 동일 확인 카드에 전달하는 기존 경로를 유지한다.
- 사용자 작품·캐논·PLAY transcript·온보딩 저장 스키마는 변경하지 않는다.

## 수용 기준

- 인기 프리셋 → 확인 카드에서 해당 scene·myRole·cast만 보이고 범용 상대 그룹이 없다.
- 함께 구상 → 확인 카드에서도 응결된 setup만 보이고 범용 상대 그룹이 없다.
- 상대를 바꾸면 선택 상태가 정확히 한 명에게만 적용되고 `이대로 시작`이 같은 상대를 사용한다.
- `이전`은 기존 `playSeedEntry`에 따라 프리셋 또는 함께 구상으로 돌아간다.
- 1280·390·320px에서 카드·상대 버튼·행동 버튼이 겹치거나 가로로 넘치지 않는다.
- focused 테스트, `bash init.sh`, Quick 페르소나 재검토와 브라우저 검증이 녹색이다.
