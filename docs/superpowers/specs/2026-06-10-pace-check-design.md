# 회차 진도 체크 + 갈림길 페이스 보강 설계 (2026-06-10 2차)

> A/B 실증(05-hunter-ab-forks.md)의 과회수 신호로 승격된 트랙. 사용자 아이디어(handoff 2026-06-10) "회차 진도 인터뷰"의 1단계(결정론 객관식) MVP + 갈림길 시드 페이스 결함 2건(P7·시드 강도) 수정.

## 문제

1. **페이스 입력 부재** — 갈림길은 "무엇을 회수할지"만 말할 수 있고 "어떤 페이스로"는 말할 수 없다. A암 ch2 검토 4건이 과회수(정답을 너무 빨리 건넴·관계 델타 앞지름)를 지적.
2. **시드 강도 일률** — deferred-stake fork 시드가 정체 여부와 무관하게 "더 미루지 않고 …결판낸다"라 비정체 상황에서도 결판을 강제.
3. **P7 — 시드 잔류** — 생성에 소비된 fork 시드가 의도 메모에 남아 다음 화에 이미 회수된 약속을 재지시.
4. **stake 문자열 드리프트** — 같은 위험이 회차마다 다른 문구로 기록되면 다른 stake 로 취급돼 회수된 위험이 fork 옵션에 잔존.

## 핵심 설계 결정

- **프롬프트 배선 0** — 의도 메모(draftPrompt)는 `produceEpisode` 에서 `freewrite` 로 LLM 프롬프트에 직접 들어간다(StoryXDesk:1751). 진도 체크 답변도 갈림길과 같은 `composeIntentWithFork` append 로 의도 메모에 합성한다. promptBuilders·storyx.mjs 미러·vite 브리지 변경 불필요.
- **결정론(LLM 0회)** — 질문·옵션·시드 모두 payoffLedger·stakesLedger 에서 결정론 도출. 서술형 LLM 인터뷰(쇼러너 대화)는 후속 2단계로 분리(갈림길 LLM 정제와 같은 묶음).

## A. 회차 진도 체크 카드 (paceInterview.ts 신설)

- **트리거(좁게)** — 연재 포맷(isSerialFormat) AND chapters.length ≥ 2 AND (ledger.isStalled OR deferred stake 2개 이상). 단편·초기 회차에는 안 뜬다.
- **질문 3개(고정)**
  1. 전제 능선 — "전제(중심 질문)가 어디까지 왔나요?" 초입/중턱/정상 직전 → 각각 페이스 시드(초입="이번 화는 토대를 쌓되 큰 reveal 은 아낀다" 류).
  2. 이번 화 페이스 — 전진/숨 고르기/회수 → 시드.
  3. 다음 큰 회수까지 — 이번 화/1~2화 안/3화 이상 → 시드("다음 큰 회수는 N화 안에 온다 — 이번 화는 그 직전 단계까지만 전진한다" 류).
- **UI** — FloatingEditor 상태 패널, 갈림길 카드 아래 `.fc-pace` 카드. 옵션 클릭 → 기존 fork 와 동일한 ref append 핸들러 재사용. 질문당 1개 선택(같은 질문의 다른 옵션 클릭 시 이전 시드 줄을 교체).

## B. 갈림길 페이스 결함 수정 (episodeBriefing.ts)

1. **시드 강도 2단(자동)** — deferred-stake fork 시드를 ledger.isStalled 에 연동: 정체="더 미루지 않고 인물의 선택과 대가로 결판낸다" / 비정체="인물의 행동으로 한 발 다가가되, 결판을 서두르지 않는다". 핀 테스트.
2. **P7** — `stripConsumedSeeds(intent)` 신설: 시드 템플릿 줄(`이번 화에서 "…"…` / `이번 화의 중심 사건은 "…"다.` / 진도 체크 시드)만 제거, 작가 자필 줄은 보존. produceEpisode 성공 경로(LLM·폴백 양쪽)에서 `setDraftPrompt(stripConsumedSeeds(...))`.
3. **stake 드리프트(보수적)** — deferred 맵 구축 시 토큰 정규화 매칭(2자 이상 토큰 Jaccard ≥ 2/3 또는 포함 관계)으로 같은 stake 로 취급, 최신 결말 우선. 별도 커밋(드랍 가능하게). 거짓 병합 가드 핀 테스트 필수.

## 검증

- 단위: paceInterview 트리거 경계·결정론·질문당 교체 / stripConsumedSeeds 보존·제거 / 시드 강도 분기 / 드리프트 매칭+거짓병합 가드.
- 라이브: #3 A암 상태에서 진도 체크 카드 렌더·클릭→메모 교체·생성 후 시드 제거 확인.
