## 1. 한 줄 정의와 판정 기준

**Dive X** — 캐릭터·배경이 주어진 상태에서 자유 대화로 몰입하다가, 일정 분량/극적 분기에서 그 대화 토막이 "한 회차(3인칭 서사)"로 응결되고, 연속성 엔진이 그것을 캐논·기억으로 고정해, 캐릭터가 그 기억 위에서 다음 대화를 이어가는 제품.

**이 프로토타입의 판정 기준은 단 하나다** — 제작자(사용자) 본인이 로컬에서 직접 돌려보고, **응결된 회차의 질감 + "캐릭터가 진짜 기억한다"는 감각**이 자기 취향 바를 넘느냐. 외부 사용자 모집·A/B·계측·PMF 통계는 전부 비목표(통과 후 별도 트랙).

이 판정 기준은 의도적이다. 세 외부 리서치가 공통으로 "수요(PMF)는 미검증"이라 결론냈고, 사용자 본인이 "Story X조차 내가 마음에 드는 첫 글이 아직 안 나왔다"고 했다. 만드는 사람이 만족하지 못하는 산출을 외부 지표로 검증하는 건 순서가 틀렸다. 먼저 손맛으로 품질을 올린다.

## 2. 합의된 결정 (브레인스토밍)

| 갈림길 | 결정 | 근거 |
|---|---|---|
| Story X와의 관계 | 별도 제품 + 공유 엔진 + 단방향 다리(C). **단 검증 단계에선 story-x 레포 안 `/dive` surface로 엔진 직접 공유** | 두 시장의 안전·비용 구조가 달라 한 앱에 욱여넣으면 둘 다 흐려짐. 엔진은 공유해야 의미. |
| 일차 차별점 | 관계/몰입 우선, **작품화 = 그 관계가 기억되고 살아남는 장치** | 관계형은 제타 본진이나, 관계형의 1번 고질병("관계가 휘발된다")을 Story X 해자(연속성)가 정확히 푼다. |
| 핵심 루프 | 자동 연대기 — 대화하면 배경에서 "우리 이야기"가 쌓임 | 마찰 제로로 관계 몰입을 안 깸. |
| 응결 트리거 | 하이브리드 — 자동 제안 + 언제든 수동 | 마찰 최소 + 통제감. |
| 회차 POV | 실시간 대화 = 1·2인칭, 응결 회차 = 3인칭 | 역할이 다름. 둘 다 필요. |
| 캐논 고정 | **승인형** — 자동 박제 금지, 사용자 승인 후 하드 캐논 | AI 오캐논화 시 신뢰 붕괴(외부 리서치 1). |
| 제품 형태 | **검증 먼저 → 가볍게·로컬·사용자 본인 판관** | 외부 리서치 3종 공통 "수요 미검증" + 사용자 결정. |

## 3. 전략적 전제 (외부 리서치 삼각측량 반영)

이 프로토타입이 짊어진 전제와 교정이다. 스펙은 이를 명시적으로 안고 간다.

- **차별점은 "비용"이 아니라 "일관성·영속·질감"이다.** digest 압축은 턴당 토큰을 평탄하게 만들지만, 1차 비용 변수는 컨텍스트 길이가 아니라 "자체 소형모델 vs 외부 프론티어 API"(10~100배). 외부 API로는 제타 비용 구조를 못 따라잡는다. 따라서 응결·기억은 **비용 절감이 아니라 일관성·영속**으로 포지셔닝한다. (검증 단계는 로컬이라 비용 무관 — 단 포지셔닝 언어를 처음부터 바르게.)
- **white space는 좁다.** "대화의 작품화" 프레임은 제타가 이미 자기 정체성으로 점유했고, 네이버웹툰 바이어스(대화→서사 텍스트, 공식 IP 한정)·뤼튼 크랙(요약 메모리)이 인접 구현을 점유했다. **방어 가능한 창은 "장편 연속성 품질 + 오리지널 + 대화→회차→캐논 폐루프"의 교집합** 하나뿐이며, 이 교집합의 상용 선례는 확인되지 않았다.
- **수요는 미검증이다.** "사용자가 진짜 주인공이 되는 1인칭 협업 서사"는 내부 딥리서치에서 명시적으로 반박(1-2)됐다 — 선두도 미구현. 차별점이자 미검증 리스크.
- **품질 의존성** — 응결 회차가 평탄하면 Story X와 같은 벽(정렬 typicality bias·조기 해소). 해법은 Dive X 코드가 아니라 Story X 품질 로드맵(Phase B 긴장·날것, 헌장 정보비대칭 레인) 쪽일 수 있다. 그 경우 갈라서 처리.

## 4. 시스템 설계 — 단위와 경계

핵심 통찰 — **Dive X 연대기는 곧 `SeriesProject`다.** 한 캐릭터와 나의 관계 = 하나의 프로젝트. 응결된 회차 = `Chapter`. 캐논·기억 = `canonFacts`/`characters`/`openThreads`. 즉 데이터 모델·응결·캐논 추출·기억 주입은 **기존 엔진을 그대로 재사용**하고, 신설은 (a) 실시간 채팅 버퍼와 (b) 그 버퍼를 회차로 응결하는 트리거/프롬프트, (c) 얇은 표면뿐이다.

### 4.1 신설 모듈

- **`src/lib/diveSession.ts`** (순수) — Dive 세션 도메인.
  - 타입 `DiveSession` — `characterId`, `projectId`(연대기), `chatBuffer: DiveMessage[]`(미응결 실시간 로그), 메타.
  - `appendMessage(session, msg)` — 버퍼에 추가.
  - `shouldSuggestCondense(session)` — 분량/턴 수 휴리스틱으로 자동 제안 여부(결정론). 임계값 상수.
  - `selectCondenseSpan(session)` — 응결할 버퍼 구간(오래된 쪽)과 잔존 구간(최근 N턴 원문 유지) 분리.
  - `applyCondenseResult(session, chapter)` — 응결 후 버퍼의 응결 구간을 비우고 회차 참조로 치환(컨텍스트 압축의 실체).
  - TDD 먼저 — `diveSession.test.ts`.
- **`src/lib/diveSeedCharacters.ts`** — 시드 캐릭터 3종(§6). `CharacterProfile` 시드 + 배경.
- **`src/components/DiveDesk.tsx`** (얇은 표면) — 채팅 뷰 + 응결 제안 칩/버튼 + 승인 다이얼로그 + 연대기 뷰. Linear 다크 토큰(`--lc-*`/`--nx-*`/`--sx-*`). `?stage=dive` 또는 `/dive` 진입.

### 4.2 재사용 (무변경 우선)

| Dive X 필요 | 기존 자산 |
|---|---|
| 회차 응결 후 캐논 추출·승격 | `commitChapter`·`extractEntityName`·`promoteCharactersFromCanon`·`linkRelationsFromCanon` |
| 기억 고정·연속성 | `storyOntology`·`continuityContract` |
| 다음 대화에 기억 주입 | `buildProjectContextDigest` (+ B3 `alwaysInclude` 핀) |
| 회차 품질 바닥 | `koreanVoiceGate` + B1 `leakGate`(`inspectLeak`) |
| 데이터 모델·영속 | `SeriesProject`·`storage.ts`(`saveProject`/`normalizeProject`) |
| LLM 호출 | 기존 dev 브리지(codex provider)·`tools/storyx.mjs` 미러 |

### 4.3 LLM 경로 (로컬)

- **`/api/dive-chat`** — 실시간 턴. 입력 = 캐릭터 카드 + `buildProjectContextDigest(project)` + 최근 N턴 버퍼. 출력 = 1·2인칭 캐릭터 응답. **경량 모델 의도**(티어링) — 로컬 codex는 단일이라 모델 파라미터만 배선하고 실제 분리는 productization.
- **`/api/dive-condense`** — 응결. 입력 = 응결 구간 버퍼 + 캐논. 출력 = 3인칭 회차 초안 + 추출 캐논 후보. **고급 모델 의도.** `produceEpisode`/`buildDraftPrompt` 계열 재사용·확장.
- 두 라우트 모두 `vite.config.ts` storyxBridge + `storyx.mjs` 미러 패턴 그대로(기존 5라우트와 동형). 키 없으면 mock 폴백.

### 4.4 데이터 흐름 (한 바퀴)

1. 사용자가 시드 캐릭터 선택 → `DiveSession` + 빈 `SeriesProject`(연대기) 생성, 캐릭터를 `characters`에 시드.
2. 대화 — 매 턴 `/api/dive-chat`(digest 주입) → 캐릭터 1·2인칭 응답 → `chatBuffer`에 누적, localStorage 영속.
3. `shouldSuggestCondense` true 또는 사용자 버튼 → `/api/dive-condense` → 3인칭 회차 초안 + 캐논 후보.
4. **품질 게이트** — 회차 초안에 `inspectLeak` + `koreanVoiceGate`. 누수면 차단/재생성.
5. **승인 다이얼로그** — 사용자가 회차·캐논 후보를 보고 승인/수정/거절. 승인 시 `commitChapter`로 회차 확정 + 캐논 승격, `applyCondenseResult`로 버퍼 압축.
6. 연대기 뷰에 회차 누적. 다음 턴부터 새 캐논이 digest로 주입 → "캐릭터가 기억한다."

## 5. 영속

- 연대기 = `SeriesProject` → 기존 `saveProject`/`loadProject`·`normalizeProject` 그대로.
- `DiveSession`(채팅 버퍼 포함) → 독립 키 `serial-story-studio/dive`에 직렬화. `parse*`/백필 패턴은 `storage.ts`의 OnboardingDraft 선례(견고 파서·구버전 null·누락 백필) 따른다.

## 6. 시드 캐릭터 3종

전연령·건전, 톤을 분산해 루프를 여러 결로 체감한다. 전부 사용자가 최소 편집 가능.

1. **무뚝뚝한 소꿉친구 (일상/로맨스 결)** — 정 많은데 표현이 서툰 동네 친구. 배경 = 같은 동네에서 자란 사이, 비 오는 날의 사소한 기억들. 관계 응결이 잘 보이는 결.
2. **떠돌이 검객 (판타지/모험 결)** — 과거를 숨긴 동행. 배경 = 함께 길을 떠난 여정, 마을마다 쌓이는 사건. 사건·약속의 캐논화가 잘 드러나는 결.
3. **밤샘 라디오 DJ (힐링/위로 결)** — 새벽 방송으로만 만나는 목소리. 배경 = 익명의 사연과 응답이 회차마다 쌓임. 정서적 연속성이 잘 드러나는 결.

## 7. 비목표 (YAGNI)

외부 사용자 모집·A/B·analytics·PMF 게이트·NSFW·결제·구독·공개 연재·창작자 마켓·캐릭터 생성 폴리시·Story X 졸업 다리·멀티캐릭터 그룹챗·모델 티어링 실분리·풀 규제 아키텍처. 전부 사용자 만족(판정 기준) 통과 후 별도 트랙. 검증 단계는 NSFW 차단 + 전연령 시드면 충분.

## 8. 외부 리서치 요약 (참조)

3종을 별도로 돌려 삼각측량했다. 공통 결론 — 제타와 무료·몰입·캐릭터 수로 싸우면 패배, 기억·영속·작품화로 측면 진입해야 함, 자체/경량 모델이 비용 전제, 폐루프의 뚜렷한 상용 리더 없음. 가장 날카로운 보고서(Claude)만 (a) 비용→일관성 교정 (b) B2C vs B2B/Story X 결합 포크 (c) 인접 선례(바이어스·크랙)로 white space 축소 (d) 영속성=법적 책임 증폭기 (e) falsification 4-게이트를 잡았다. 학술 근거 — TimeChara(GPT-4o 미래질문 46~51%)·LoCoMo(인간 대비 56%·시간추론 73% 뒤)·CharacterBench·HaluMem 등이 "장기 기억·캐릭터 일관성"이 미해결임을 보임 — Story X 연속성 엔진의 기술 정당성.

## 9. 리스크

- **UX 템포 충돌** — 실시간 채팅 템포 vs 3인칭 서사 읽기 템포는 인지 피로도가 다르다. 응결 회차를 *언제·어떻게* 끼우느냐가 "마법 vs 흐름 끊김"을 가른다. 자동 연대기의 숨은 급소.
- **응결 품질 평탄함** — §3 품질 의존성. 막히면 Story X 품질 로드맵으로 분기.
- **수요 미검증** — 핵심 가설 자체. 이 프로토타입의 존재 이유가 그걸 사용자 손으로 먼저 깨는 것.
- **법적 증폭(deferred)** — 영속 저장은 한국 아청법 표현물 리스크를 키운다. 검증 단계는 전연령·비공개·로컬이라 보류, 외부 공개 전 day-1 아키텍처로 재소환.

## 10. 검증(Definition of Done — 프로토타입)

- `npm test` 전체 녹색(신설 `diveSession.test.ts` + 기존 회귀), `npm run build` 성공.
- 새 생성 동작은 `storyEngine.test.ts`/`diveSession.test.ts`를 먼저 갱신(TDD).
- 로컬 라이브 — 시드 캐릭터로 대화 → 응결 제안 → 승인 → 회차·캐논 확정 → 다음 대화에서 그 캐논이 digest로 주입돼 캐릭터가 회수하는 한 바퀴를 사용자가 직접 확인.
- Linear 다크 토큰 유지. 한국어 UI 간결.
- 판정 기준(사용자 만족)은 코드 DoD가 아니라 dogfooding 루프의 목표 — 반복 대상.
