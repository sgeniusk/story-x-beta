# 실사용 창작자 10인 풀 라이브 실증 테스트 — 실행 Plan

> **에이전트 워커용** — 이 plan은 코드 구현이 아니라 **dev 앱 직접조작 테스트**다. 단계는 체크박스(`- [ ]`)로 추적한다. spec은 `docs/superpowers/specs/2026-06-07-persona-live-test-design.md`.

**Goal** — 실사용 창작자 10인 페르소나로 스토리X를 직접 조작해 이야기 생성·반복수정·(장편)완권 연속성을 실증 측정하고, 종합 리포트와 새 제작 계획의 근거를 만든다.

**Architecture** — 로컬 dev(provider codex) + Playwright 직접조작. 멀티세션 순차(S0~S15). 페르소나마다 라이브 로그·스크린샷·6축 평가를 남긴다.

**Tech Stack** — Vite dev(127.0.0.1:5173) · Playwright MCP · storyx codex provider · floating editor(기본).

## Task 0 — S0(오늘) #1 파일럿: 웹소설 장편 도입 1~3화

**페르소나 #1** — 28세 남, 출퇴근 지하철에서 웹소설 보는 직장인. 정리해고 충격 후 "나도 한 번 써볼까" 첫 연재 도전. 회귀·성장 판타지. 말투는 캐주얼, 웹소설 트로프(회귀·먼치킨·사이다)에 밝고 문장은 거칠다.

**freewrite 입력안(실제로 이 텍스트를 인터뷰에 입력)**
> 회사에서 정리해고 통보받은 날 퇴근길에 죽었다. 눈을 뜨니 10년 전, 헌터 각성 시험 전날이다. 그때 나는 최하위 등급으로 각성해서 평생 잡몹만 잡다 죽었다. 근데 이번엔 안다. 어떤 게이트가 언제 열리는지, 누가 영웅인 척하다 배신하는지, 어떤 스킬이 사기인지 전부. 이번 생은 다르게 산다. 최약체에서 시작해 정점까지.

**기록물** — `docs/reviews/2026-06-07-persona-live-test/01-webnovel-regression.md` + 스크린샷 `01/`.

- [ ] **S0-1 진입·캡처** — Playwright로 `127.0.0.1:5173` 접속. 랜딩 스크린샷. floating 기본인지 확인(`?editor=classic` 아님).
- [ ] **S0-2 매체/포맷 선택** — 소설 → 장편. 화면 snapshot으로 실제 버튼/플로우 파악 후 클릭. 매체 선택 화면 캡처.
- [ ] **S0-3 인터뷰 입력** — freewrite 입력안 텍스트를 인터뷰 자유서술에 입력·제출. 인터뷰가 페르소나의 어휘를 받아쓰는지(이전 강점) 관찰·기록.
- [ ] **S0-4 1화 초안 생성** — 초안 생성 실행(codex 실호출). 생성 소요 시간 측정. 결과 본문·beats·캐논 캡처. **생성 품질** 1차 메모(웹소설 훅·사이다·문장).
- [ ] **S0-5 검토(마진 5명)** — 전체 검토 실행. 5명(쇼러너·캐릭터·세계·장르·연속성) 의견 도착 확인. 통과/수정/차단 분포 기록. 검토 소요 측정.
- [ ] **S0-6 수정 사이클 3~4회** — 검토 의견 반영(`acceptMarginDiff`) + 본문 직접 편집을 페르소나 입장에서 반복. 매 회 "무엇을 고침 / 시스템 반응 / 연속성 영향" 기록. **수정 반응성** 관찰.
- [ ] **S0-7 2화 생성(회차 누적)** — 다음 회차 진행. 2화가 1화의 캐논·주인공 능력·등장인물을 이어받는지 확인(회귀 설정·각성 등급 등). 어긋나면 그대로 기록(연속성 결함은 드러낸다).
- [ ] **S0-8 3화 생성 + 연속성 확인** — 3화 누적. 1~3화 통독으로 주인공 이름·능력·말투·세계 규칙 드리프트 점검. **캐릭터 추적표** 1차 작성(이름·말투·능력·관계).
- [ ] **S0-9 6축 평가 기록** — 생성품질·수정반응성·연속성·매체적합성·UX마찰·상업성 각 1~5점 + 종합 통과/수정/차단 + 근거. UX 마찰(막힘·혼란) 빠짐없이.
- [ ] **S0-10 파일럿 메타 측정** — 1화당 평균 소요(생성+검토+수정), 프로토콜 병목(느린 단계·자동화 까다로운 조작), Playwright 조작 난점 기록 → S1~S9 완권 세션 수 추정 갱신.
- [ ] **S0-11 상태 동기화** — `progress.md` 병행 트랙에 본 테스트 추가, `session-handoff.md` 맨 위에 "S0 파일럿 결과 + 다음=#1 완권(S1~S3) 또는 S1 착수" 인계. `feature_list.json` 필요 시 마일스톤 추가.

## S1~S15 세션 개요 (파일럿 측정 후 plan 확정)

- **S1~S3** — #1 웹소설 장편 **완권 완주**(~20~25화). 회차 누적·5화/12화 의도적 연속성 부하·캐릭터 추적·완권 통독.
- **S4~S6** — #2 로맨스판타지 장편 완권 (장기 감정선 일관성).
- **S7~S9** — #3 헌터물 장편 완권 (**다수 등장인물 캐릭터 일관성 집중**).
- **S10** — #4 미스터리 중편 (**용의자·탐정 캐릭터 일관성 집중** + 복선/반전).
- **S11** — #5 순문학 단편 · #6 SF 단편 (각 1편 + ~10회 수정).
- **S12** — #7 인스타툰 4컷 · #8 개인 에세이 (매체 1급 시민 여부·사실 보호).
- **S13** — #9 동요읽기 · #10 사회과학 논문 (낭독성·주장근거/인용).
- **S14** — **종합 실증 리포트** `docs/story-x-10-live-creator-report.md` (6축 종합·반복 발견·P0/P1).
- **S15** — 리포트 위에서 **새 제작 계획** 확정(이전 12인/20인·thesis·PRD 종합, master plan 정합).

세션 경계는 회차 단위로 끊고 매번 handoff 갱신. 장편은 1명당 ~3세션 추정(S0 측정으로 확정).
