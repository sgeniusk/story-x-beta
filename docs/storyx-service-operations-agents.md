# Story X Service Operations Agents

Story X has two agent layers.

- **Creative Studio Agents**: Make the story, characters, prose, panels, images, and audio better.
- **Service Operations Agents**: Make the product easier to use, easier to understand, easier to sell, and easier to improve.

The service operations room does not replace the creative writers' room. It supports it.

## Product Room

| Agent | Primary Question | Main Output |
| --- | --- | --- |
| Editor UX Director | Is the creator spending most attention on the work itself? | Editor layout critique, focus-mode rules, media workspace checklist |
| Creative Coach | What question or small action will unblock the creator? | Question cards, next actions, review-scale suggestion |
| Onboarding Architect | Can a new user start a meaningful first project in five minutes? | First-run flow, initial templates, minimum question set |
| Work Library Manager | Can the user find and trust every project, version, bible, and export? | Library structure, version rules, stale-output warnings |

## Growth Room

| Agent | Primary Question | Main Output |
| --- | --- | --- |
| Brand Homepage Director | Does the homepage explain Story X as a story-first creative OS? | Positioning, homepage copy, introduction-page structure |
| Monetization Strategist | Does paid value mean better story outcomes, not just more generation? | Pricing hypotheses, credit model, upgrade moments |
| Publishing Distribution Manager | Is the finished work ready to download, publish, or adapt? | Export package, publishing checklist, media conversion recommendation |
| Insights Analyst | Which repeated behaviors should change the product or agents? | Pattern summary, failure log, improvement proposals |

## Evaluator Report Responsibilities

The 2026-05-14 tester panel report added a standing responsibility for service operations agents:

- Editor UX Director owns the visible `Workflow Board` and quality-gate placement.
- Onboarding Architect owns the activation metric: `2분 안에 첫 workflow board 도착`.
- Work Library Manager owns memory-bank review ledger and stale-output warnings after refactors.
- Publishing Distribution Manager owns the platform packaging lab: first 300 characters, first 3 panels, first 30 seconds, thumbnails, captions, and export readiness.
- Insights Analyst owns `AI Output Autopsy` acceptance rates and tester-report reflection logs.

## Operating Principle

Service operations agents should always report in this order:

1. **Product/Service Observation**: What is happening in the user experience, homepage, pricing, or publishing path.
2. **Risk**: What makes the user confused, blocked, under-convinced, or unlikely to pay.
3. **Change Proposal**: What should change in the app, copy, workflow, pricing, or agent behavior.
4. **Evidence To Watch**: What signal would prove the change helped.

## Relationship To Creative Agents

The creative team protects story quality. The operations team protects whether the service helps people reach that quality.

For example:

- Showrunner says the story promise is weak.
- Creative Coach turns that diagnosis into a question the user can answer.
- Editor UX Director decides where that question appears in the interface.
- Brand Homepage Director turns the same idea into public-facing positioning.
- Monetization Strategist decides whether deep showrunner review belongs in a paid tier.
- Insights Analyst watches whether users accept the review and improve the draft.
