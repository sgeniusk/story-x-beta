### 구조적 이유
미스터리·로판·스릴러의 **반전(reveal)은 본디 부정형으로 서술된다** — "범인은 X가 아니다", "권한자는 레나가 아니며 루시안이…". 이런 팩트가 캐논에 쌓이면 `copulaNegatedNouns`가 그 술어명사(대개 인명)를 "부정된 명사"로 잡고, `findReversalMatch` 둘째 루프가 **claim이 그 명사를 언급하기만 하면**(주어·주제로 등장해도) 반전으로 판정한다. claim이 그 명사를 **술어로 단정**하는지는 안 본다.

```
// findReversalMatch 둘째 루프 (continuityContract.ts:404)
for (const noun of sourceCopula) {
  if (expandedClaim.has(noun) && !claimCopula.has(noun)) return source;  // ← 언급만으로 발화
}
```
- 의도된 케이스(첫째 루프) = claim "윤민서는 형사**가 아니**"라 vs source "…형사이며" → 진짜 반전(정당).
- 누적 오발화(둘째 루프) = source "권한자는 레나**가 아니며**" vs claim "레나는 …경고했다"(레나를 술어로 부정하지 않고 주어로 언급) → **가짜 반전**.

## 판단
- **지배적 문제는 recall 갭이 아니라 정밀도 붕괴다.** 핸드오프 불변식 "정밀한 직접 모순만 · 오탐 0"이 누적에서 위반된다.
- #32의 recall 보강(계사부정)이 **정확히 이 오탐을 유발**했고, 1화 격리 테스트가 이를 가렸다. "recall 올리려다 오탐 내지 말 것"(손대지 말 것) 경계를 누적에서 넘은 것.
- 멸문-miss가 시사하듯 **누적은 초기 하드팩트를 reveal로 완화**하기도 한다 — 게이트가 "타임스탬프 없는 팩트 풀"로 캐논을 다루는 한, 오래된 팩트와 그 반전이 같은 풀에서 서로를 오염시킨다.
