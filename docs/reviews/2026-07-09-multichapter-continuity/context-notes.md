# 컨텍스트 노트 — 멀티회차 누적 연속성 검증 라운드 (2026-07-09)

## 목표
핸드오프 1순위(사용자 강조) — 예비비행은 각 소재 1화만 생성해 캐논 축적 대비 충돌을 못 밟았다. 회차가 쌓일 때 연속성 게이트가 실제로 작동하는지, #32 보강이 **누적 상태**에서도 recall/정밀도를 유지하는지 관찰.

## 방법 결정 (사용자 답변 2건)
- **결정론 하네스 먼저 + 예비비행 소재 재사용**.
- 두 답변을 2단계로 조화 — ① 결정론 단계는 저장소에 이미 있는 **실제 누적 프로젝트**(23화·91팩트 로판 백업)로 즉시 실측(생성 0·재현 가능). 예비비행 1화 소재엔 누적이 없어 결정론 하네스에 부적합했기 때문. ② 실제 codex 이어 생성(유기적 드리프트)은 **예비비행 #6 스릴러 소재**로 후속(미착수).

## 왜 ch23 백업이 이상적 픽스처인가
- 91개 캐논 팩트 × 23화 = 진짜 누적. 손수 합성 아님.
- 캐릭터 `forbiddenContradictions`가 전부 비어 있음(0) → 발화하는 **유일한 메커니즘이 자동 의미 게이트**(classifyCanonChange). 명시 규칙 경로가 결과를 오염시키지 않음 → 핸드오프가 겨냥한 자동층을 순수 측정.

## 산출물
- `gate-accumulation.ts` — 정밀도(재진술 91 + 정합신규 4) + recall(직접모순 5) 실측.

- `findings.md` — 정량 결과 + 근인 + 권고.

## 핵심 결과 (findings.md 상세)
- **정밀도 붕괴** — 재진술 FP 53/91, 정합신규 3/4. 1화 격리(#32)의 오탐 0이 누적에서 무너짐.
- **근인** — ~51/53이 #32 계사부정 매처 과발화(reveal 팩트 "X가 아니며"가 그 엔티티 언급을 전부 위반 처리).
- recall 3/5(부차). 멸문-miss는 후반 캐논이 이미 완화해 정당한 non-block일 수 있음.

## 불변식 유의
- 이 발견은 "손대지 말 것"(정밀도 우선·오탐 0·recall 올리려다 오탐 금지) 영역을 직접 건드린다. 수정은 사용자 결정 후 브레인스토밍→spec→TDD.
- 결정론 유지 — LLM·사전 금지. classifyCanonChange는 순수 코드.
