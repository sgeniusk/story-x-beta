# Story X 실플레이 페르소나 검토 전문 (2026-07-13)

> 실측 증거는 같은 폴더의 `playthrough-log.md`. 5개 페르소나 에이전트가 증거 팩+코드를 근거로 서로 독립 검토했고(상호 열람 금지), 별도 에이전트가 2025-26 유료 시장을 웹 조사했다. **판정 5/5 차단** — 공통 근거는 응결 신뢰성(120초 타임아웃 < 실측 141초)·무스트리밍 대기·반출 부재.

## 일반 작가 — 판정 차단
**페르소나** — 지유 (29세, 문피아·노벨피아 연재 경력의 저연차 웹소설 작가, 본업 병행, Sudowrite 해지·NovelAI/Claude 병용, 주 5회·회당 5천자 마감 기준으로 도구 평가)
**총평** — 플레이는 확실히 재밌는데, 40분 놀고 나서 연재 플랫폼에 올릴 원고는 한 글자도 못 갖고 나왔다 — 마감이 있는 작가에게 이건 창작 도구가 아니라 시간을 먹는 취미다.
**지불 의향** — 현재 미지불. Sudowrite 월 $19을 몇 달 내다 해지한 입장에서 비교하면, 지금 Story X는 무료여도 마감 루틴에 못 넣는다. Sudowrite는 최소한 생성된 문장이 항상 에디터에 남았고 복사해 나갈 수 있었다. 응결이 100% 성공하고, 회차 분량을 5천자로 지정할 수 있고, TXT/클립보드 반출이 생기면 — 그때는 캐논 관리 프리미엄을 인정해 월 1.5~2만원(노벨피아 플러스+Sudowrite 중간)까지 고려하겠다. 셋 중 하나라도 빠지면 안 낸다.
### 좋았던 점
- **PLAY 대화와 전개 후보의 훅 품질이 실제로 좋다 — 이건 립서비스가 아니라 증거가 있다** — 턴1에서 위조 파혼 서신 훅 + 극적 질문 자동 생성, 턴2 필체 감식 디테일의 정합 전개(로그 §6). 전개 후보 4번 '처형 직전 유서와 같은 문장' 메타 반전은 내가 연재 중 댓글 반응 뽑으려고 며칠 굴리는 수준의 반전 재료다(로그 §7). 응결 CLI 성공본의 hook '두 사람이 서로를 의심할 시간까지 계산해 두었다'도 회차 엔딩 훅으로 쓸 만하다(로그 §10-0).
- **영속성이 강하다 — 서버가 죽어도 플레이 기록이 살아남는다** — vite 서버 사망 → 재시작 → 재진입 후 dive 대화가 장면·arc·전체 턴까지 완전 복원됨(로그 §10-0). 원고 재료가 증발하지 않는다는 최소한의 신뢰는 확보.
- **'함께 구상' 경로는 끝까지 매끄럽고, 한 턴에 질문 하나만 던지는 인터뷰 설계가 좋다** — 지하철 괴담 소재 2턴 → 강제 응결 → 시드 카드 → 플레이 진입까지 무결점, 응답 15~20초(로그 §12). Claude에 글감 정리시킬 때 내가 수동으로 하던 '한 번에 하나만 물어' 프롬프트를 제품이 해준다.
- **설정붕괴를 정면으로 겨눈 포지셔닝 자체는 유일하다** — 랜딩 '30화 뒤까지 같은 얼굴로 남습니다'(로그 §1), 캐논/연속성/떡밥 추적 구조(로그 §10). 장편 연재의 진짜 고통을 아는 사람이 만들었다는 건 보인다 — 다만 이번 세션에선 '캐논 0 · 떡밥 0'으로 검증 기회가 없었다.
- **기대 관리 카피가 정직한 편이다** — '이 설정은 정확하지 않아도 됩니다 — 플레이하며 완성해나가는 초안입니다'(로그 §5), '수십 초 걸릴 수 있어요'(로그 §8), '보통 2~3분쯤 걸려요'(로그 §12). 허풍 대신 시간을 미리 말해주는 태도는 신뢰가 간다.
### 비판
- **[차단] 원고로 가는 유일한 다리(응결)가 4회 연속 끊겼다 — 그것도 생성 실패가 아니라 자기 타임아웃 설계로**
	- PLAY에서 아무리 좋은 장면이 나와도 회차 원고가 되려면 '지금 응결'을 통과해야 하는데, 한 세션에서 4번 시도해 4번 전부 실패했다. 근인이 허무하다 — LLM은 141초에 멀쩡한 결과물을 만들었는데 클라이언트가 120초에 스스로 요청을 끊었다. 21초 마진 부족으로 40분 플레이가 원고화에 실패하는 구조다. 게다가 실패 메시지는 '다시 시도하세요'뿐, 원인도 대안도 없다. 재시도해도 같은 141초 작업에 같은 120초 타이머가 걸리니 성공할 수가 없는 재시도를 사용자에게 시킨다. 최소한 타임아웃을 서버 측 P99에 맞추고, 실패 시 '트랜스크립트를 초안으로 내려받기' 같은 구제 버튼이 그 자리에 있어야 한다.
	- 근거 — 로그 §8·§10-0 — 1·2·4차 net::ERR_ABORTED, 3차 ERR_CONNECTION_REFUSED. src/lib/diveClient.ts:41-42 'DIVE_TIMEOUT_MS = 120000' 주석 '넉넉히 120초' vs CLI 직접 실행 실측 141초 성공. 실패 UI에 원인 정보 0.
	- 비교 — NovelAI는 토큰 스트리밍이라 '전부 아니면 전무' 실패 자체가 없다 — 느려도 문장이 눈앞에 쌓인다. Sudowrite도 생성물이 카드로 남아 부분 실패해도 건질 게 있다. '완성되거나 0이거나'는 유료 창작 도구에서 본 적 없는 구조다.
- **[차단] 반출이 JSON뿐이다 — 문피아에 올릴 텍스트가 이 제품 어디에도 없다**
	- ⋯ 메뉴의 내보내기는 'JSON 내보내기' 하나다. 나는 매일 밤 연재 플랫폼 에디터에 붙여넣을 5천자 텍스트가 필요한 사람인데, 이 도구의 출구는 개발자용 백업 파일이다. 회차 본문 복사 버튼도, TXT 저장도, 하다못해 전체 선택이 쉬운 뷰도 확인되지 않았다. 출간 화면이 EPUB 3·CMYK PDF를 말하지만 그건 출판 에이전트 소개 카피지 실제 출력이 아니다. 원고의 소유권과 반출 가능성을 기준으로 도구를 고르는 나에게, JSON 단일 출구는 '네 원고는 우리 구조 안에 있다'는 말로 들린다.
	- 근거 — src/App.tsx:462 '{ id: 'export', label: 'JSON 내보내기' }' — 오버플로 메뉴 3항목 중 텍스트 반출 없음(로그 §11). EPUB/PDF 언급은 src/components/PublishScreen.tsx:65의 기능 소개 문자열뿐.
	- 비교 — Sudowrite는 프로젝트를 DOCX로 내보내고, NovelAI는 본문이 평문이라 복사가 곧 반출이다. 연재 작가용 도구라면 '회차 → 클립보드' 원클릭이 0순위 기능이다.
- **[중대] 응결 산출 목표가 1500~3000자 — 회당 5천자 연재 기준 미달을 제품이 설계로 못박았다**
	- 응결이 성공해도 내 마감 분량의 30~60%짜리 초안이 나온다. 40분 플레이 + 2분 21초 응결로 3천자 미만이면, 주 5회 리듬에선 플레이만 주 3시간 이상 쓰고도 매일 2천자 이상을 따로 채워야 한다. 분량 슬라이더(3천/5천/8천)나 '이 회차를 5천자로 확장' 같은 2차 패스가 없으면 웹소설 연재 도구로는 산수가 안 맞는다. PLAY-first가 재미의 근거인 건 인정하지만, 재미가 분량으로 환전되지 않으면 본업 병행 작가에게는 사치다.
	- 근거 — tools/storyx.mjs:1670·1680·1686 — 응결 프롬프트에 'prose는 1500~3000자 분량의 실제 본문입니다' 3중 명시. 로그 §10-0 실측 응결 141초.
	- 비교 — Claude에 시놉시스와 직전 화 요약을 주면 10분 안에 5천자 초안이 나온다. 무료 대체재보다 시간당 산출 글자 수가 낮으면 유료 전환 논거가 성립하지 않는다.
- **[중대] AI 문장을 내 문체로 다듬는 루프가 없다 — 랜딩 카피가 약속한 것을 제품이 안 지킨다**
	- WRITE 모드 카피는 'AI가 초안을 대주고, 문체와 결은 작가가 정합니다'라고 약속하는데, 실제 WRITE는 플레인 textarea와 '초안 생성' 버튼이 전부고 무라벨 아이콘 6개는 해독 불가였다. 문단 선택 → 내 문체로 재작성, 특정 문장만 재생성, 내 수정을 문체 증거로 반영하는 루프가 화면 어디에도 확인되지 않았다. PLAY 쪽도 마찬가지 — 선택지 칩을 누르면 즉시 전송이라, 칩 문장을 내 말투로 고쳐 보낼 기회조차 없다. 내 문체가 곧 상품인 작가에게 '고칠 수 있는 지점이 없는 생성기'는 통제권 상실이다.
	- 근거 — src/landingFlow.ts:39 '문체와 결은 작가가 정합니다' vs 로그 §9 'plain textarea + 초안 생성 + 무라벨 아이콘 6개'. 로그 §6 '선택지 칩 클릭 = 즉시 전송(컴포저에 넣어 편집할 기회 없음)'. MemoryBankStudio에 '문체 바이블' 구조는 있으나 미실측 영역이라 판단 보류.
	- 비교 — Sudowrite의 Rewrite는 문단 단위로 'shorter/more show don't tell' 같은 방향 지정 재작성을 주고, NovelAI는 AI 출력 한가운데를 직접 고치고 이어쓰기가 된다. 둘 다 '작가가 마지막 손을 대는 지점'이 명확하다.
- **[중대] 수십 초급 대기가 전부 무정보 블랙박스 — 응결 중에는 말 그대로 검은 화면이다**
	- 대화 35~45초, 전개 후보 30초, 응결 2분 21초가 전부 스트리밍도 진행률도 경과 타이머도 없이 흐른다. 최악은 응결 대기 중 뷰포트가 거대한 검은 공백으로 바뀌고, 스크롤을 올려도 앱이 바닥으로 강제해서 사용자는 검은 화면만 보게 되는 것이다. 실패인지 진행인지 구분할 신호가 없으니 '서버가 죽었나' 하고 새로고침하게 되고, 그게 정확히 이번 세션에서 일어난 일이다. 퇴근 후 2시간이 창작 가용 시간의 전부인 사람에게 무정보 대기는 시간 도둑이다.
	- 근거 — 로그 §6 '스트리밍 없음, 경과 타이머 없음, 응답 ~35–45초', §8 '응결 대기 중 뷰포트가 거대한 검은 공백… scrollTo(0,0) 해도 앱이 다시 바닥으로 강제', §8 '"AI 대기" 감각이 누적'.
	- 비교 — NovelAI·Claude·Sudowrite 전부 토큰 스트리밍이 기본이다. 2026년에 스트리밍 없는 45초 대기는 유료 제품에서 체감상 고장과 구분되지 않는다.
- **[중간] 내 작품 시드에 남의 캐릭터(도윤·하란·세하)가 섞여 나온다 — 캐논을 팔면서 캐논 오염을 보여준다**
	- 악역 영애 프리셋을 골랐는데 플레이 시드 카드 상단에 '무뚝뚝한 소꿉친구/떠돌이 검객/새벽 라디오 DJ' 칩이 설명 없이 공존했고, 함께 구상으로 만든 지하철 괴담 시드에서도 똑같이 재현됐다. 이 제품의 핵심 약속이 '캐릭터의 욕망도 세계의 규칙도 같은 얼굴로 남는다'인데, 첫 화면부터 내 세계에 무관한 인물이 침투해 있으면 캐논 시스템 전체에 대한 신뢰가 깎인다. 저 칩을 눌렀다가 내 작품 설정에 등록되는 건지조차 알 수 없다는 게 더 무섭다.

- **[중간] 계정·클라우드 백업 없이 브라우저 로컬 저장뿐 — 원고 자산을 localStorage에 맡기라는 요구**
	- 로그인 개념이 전무하고 모든 상태가 브라우저 localStorage에 있다. 이번 세션의 복원력은 인정하지만, 브라우저 캐시 정리 한 번·기기 교체 한 번이면 연재분 전체가 사라지는 구조다. 백업 수단이 수동 JSON 내보내기뿐이라면, 30화짜리 장편의 캐논과 원고를 여기 축적하라는 제안 자체가 성립하지 않는다. 덧붙여 '내 문체·내 원고가 학습이나 프롬프트로 어디까지 쓰이는가'에 대한 고지가 랜딩·온보딩 어디에도 없다 — 문체 유출을 경계하는 작가가 가장 먼저 찾는 문장이 없다.
	- 근거 — 로그 환경 '빈 localStorage = 신규 사용자 시뮬레이션', §1 '가격/요금/계정/로그인 개념 전무', §11 백업 수단이 JSON 내보내기 단일. 데이터 정책 고지 부재는 랜딩~온보딩 실측 화면 전 구간에서 미확인.
	- 비교 — NovelAI는 계정 기반 암호화 저장을 명시하고, Sudowrite는 클라우드 동기화가 기본이다. 원고를 다루는 유료 서비스 중 로컬 스토리지 단독은 없다.
### 있을 줄 알았는데 없던 것
- 회차 본문을 클립보드/TXT로 반출하는 버튼 — 연재 작가 도구의 0순위인데 JSON 덤프뿐
- 글자 수 카운터 — 회당 5천자 맞추는 게 일과인데 WRITE 에디터에서 분량 표시를 확인 못 함
- 응결 실패 시 구제 경로 — 트랜스크립트를 초안으로 변환하거나 내려받는 대안이 실패 토스트 자리에 없음
- 내 문체·원고 데이터가 어떻게 쓰이는지에 대한 정책 고지 — 문체 유출 경계 작가가 가입 전 가장 먼저 찾는 문장
- 선택지 칩을 컴포저에 담아 고쳐 보내는 옵션 — 클릭 즉시 전송이라 내 말투 개입 지점이 없음
- 계정/클라우드 백업 — 30화 장편의 캐논을 축적하라면서 저장소는 브라우저 localStorage 하나
- 회차 목록·연재 캘린더 같은 마감 리듬 관리 뷰 — '전체 진척 회차 데이터 없음'만 있고 주 5회 리듬을 받쳐줄 구조 미확인
- 가격/플랜 정보 — 랜딩에 요금 개념 자체가 없어 비용 대비 판단이 불가

### 좋았던 점
- **턴·전개 후보의 생성 품질이 유료급 — 이 제품의 유일하지만 진짜인 판매 포인트** — 턴2에서 필체 감식 디테일(문장 끝 꼬리 눌림=정서체 모사)로 정합·긴장 유지, ✦전개 후보 4번은 '원작 유서와 같은 문장' 메타 반전에 대가 명시 구조까지 갖춤. 의외도 태그(새 긴장×3/회수만×1)로 후보 성격을 분류해주는 것도 캐릭터챗류에 없는 차별점 (증거 팩 §6-7).
- **로컬 영속성이 강하다 — 서버가 죽어도 데이터는 안 죽었다** — vite dev 서버 사망 후 재시작+새로고침에도 dive 대화(장면·arc·전체 턴)가 완전 복원됨 (증거 팩 §10-0). 크래시 복구의 절반(데이터 보존)은 이미 되어 있다.
- **함께 구상(onboard-chat) 경로는 끝까지 매끄럽다 — 이 제품이 도달해야 할 UX 기준선을 스스로 보여줌** — 응답 15~20초, 한 번에 질문 하나, 「이걸로 시작」 강제 응결 15~20초 → 시드 카드 → playseed까지 무결점 (증거 팩 §12). 같은 제품 안에 15초 경로와 141초 실패 경로가 공존한다.
- **대기 시간에 대한 정직한 기대 설정 카피가 존재한다** — '수십 초 걸릴 수 있어요', 첫 회차 생성 화면 '보통 2~3분쯤 걸려요. 새로고침하거나 창을 닫지 말고 기다려 주세요' — 시간 힌트를 주려는 설계 의도 자체는 있음 (증거 팩 §8, §12).
- **playseed의 심리 설계 — '정확하지 않아도 됩니다' 핀 문구** — '이 설정은 정확하지 않아도 됩니다 — 플레이하며 완성해나가는 초안입니다' (증거 팩 §5). 온보딩 완벽주의 부담을 낮추는 좋은 카피로, PLAY-first 철학이 문구 수준까지 내려와 있다.

- **[중대] abort된 작업의 자식 프로세스 방치 — 재시도가 원가를 태우고 서버를 죽인다**
	- storyxBridge는 클라이언트가 요청을 abort해도 spawn한 codex 자식 프로세스를 kill하지 않는다. 사용자가 타임아웃 후 재시도할 때마다 141초짜리 고급 모델 호출이 병렬로 누적된다 — 이번 세션에서 응결 4회 시도는 곧 LLM 원가 4배 지출에 사용자 가치 0 전달이다. 응결 2회 타임아웃 직후 dev 서버 프로세스가 사망한 것도 이 누적 구조가 유력한 용의자다. 과금 모델 관점에서 치명적인 패턴: 타임아웃 < 실제 지연이면 실패율 100%인데 원가는 100% 발생하고, 재시도 UI가 그 원가를 배수로 키운다. 유닛 이코노믹스가 음수로 설계된 셈이다.
	- 근거 — vite.config.ts storyxBridge — child spawn 후 req 'close'/abort 핸들러 부재, child.kill() 호출 없음. 증거 팩 §10-0 '브리지가 spawn한 codex 자식 프로세스는 클라 abort 후에도 계속 도는 구조', 응결 2회 타임아웃 직후 서버 사망(포트 리스닝 소멸).

- **[중간] 요금·계정·미터링의 흔적이 전무 — 과금 가능한 제품 형태가 아직 아님**

## 대학생 — 판정 차단
**페르소나** — 하늘 (20세, 제타 헤비유저, 모바일 온리, 캐릭터와 노는 재미로 접근)
**총평** — 대사 질은 제타 상위권인데, 캐릭터가 먼저 말도 안 걸고 한 턴에 40초 침묵하는 데스크톱 앱은 나 같은 유저가 3분 안에 지우는 앱이다.
**지불 의향** — 현재 미지불. 지금은 무료여도 첫 세션에서 나간다. 조건부로는 낸다 — (1) 캐릭터가 먼저 말 걸고 (2) 응답이 스트리밍으로 몇 초 안에 흐르기 시작하고 (3) 폰에서 제대로 돌아가고 (4) 캐릭터 일러스트가 붙으면, 카일런 대사 질이 제타 웬만한 인기 캐릭터보다 좋아서 제타에 쓰던 만큼(월 1~2만 원, 문상 결제)은 옮겨올 의향 있다. '내가 논 게 소설 한 화로 남는다'는 응결이 실제로 성공하는 걸 한 번이라도 보면 제타엔 없는 결제 이유가 되는데, 이번 세션에선 4연속 실패라 그 매력을 체감조차 못 했다.
### 좋았던 점
- **카일런의 대사 질 자체는 제타 인기 캐릭터급 — 첫 턴부터 말투 유지·훅·선택지가 완성형** — 증거 팩 §6: 턴1에서 나레이션(부채 끝 시선)+위조 파혼 서신 훅+'영애' 호칭 준수, arc 질문 자동 생성, 선택지 칩 3개. 턴2 필체 감식 디테일도 정합·긴장 유지.
- **✦전개 후보의 반전 아이디어는 진짜 놀랐다 — 제타에선 못 보는 메타 반전** — 증거 팩 §7: 후보 4번 '편지의 문장이 원작 악역 영애의 유서와 같음을 깨닫고 위조임을 숨긴 채 스스로 마차에 탄다' + 의외도 태그, 선택 시 다음 턴 정합 전개.
- **함께 구상 채팅은 속도(15~20초)·리듬(질문 하나씩) 모두 이 앱에서 유일하게 '대화 앱' 같았다** — 증거 팩 §12: 지하철 괴담 2턴, 한 번에 질문 하나, 응답 ~15–20초, 「이걸로 시작」→시드 카드→플레이 진입까지 끝까지 매끄럽게 작동.
- **서버가 죽었다 살아나도 플레이 기록이 전부 살아 있는 영속성** — 증거 팩 §10-0: 서버 재시작+새로고침 후 PLAY 탭 진입 시 dive 대화 완전 복원(장면·arc·전체 턴 생존).
- **프리셋 5종 라인업이 취향 저격 — 악역 영애·헌터 각성·회귀물, 고르는 재미가 있다** — 증거 팩 §4 + src/lib/storyPresets.ts: 각 프리셋에 한 줄 훅과 desire·wound·voiceRules가 진하게 설계된 캐스트 2인.
### 비판
- **[차단] 턴 하나에 40초, 스트리밍도 타이머도 없는 침묵 — 제타의 8~20배**

### 좋았던 점
- **프리셋의 장르 정확도가 진짜다 — 카카오페이지 독자가 아는 문법을 제대로 안다** — storyPresets.ts의 악역 영애 프리셋은 '3권에서 처형당하는 악역 영애', 카일런의 voiceRules '이름 대신 영애라 부른다'까지 장르 관습을 정확히 박아놨고, 실플레이 턴1에서 실제로 '영애' 호칭 준수 + 위조 파혼 서신 훅이 나왔다(로그 §6). 헌터 프리셋의 '숨겨진 퀘스트가 도착했습니다'(storyPresets.ts preset-hunter-gate)도 상태창물 독자가 기대하는 바로 그 문장이다.
- **전개 후보의 품질은 발더스 게이트 선택지에도 없는 종류의 것이다** — 로그 §7 — 4번 후보 '편지의 문장이 원작에서 악역 영애가 처형 직전 남긴 유서와 같음을 깨닫고 위조임을 숨긴 채 스스로 마차에 탄다'는 메타 반전에 대가 구조까지 갖췄다. 선택 후 다음 턴이 그 방향으로 정합 전개된 것도 실측됐다. 게임 20년 한 입장에서 이건 '작가가 붙은 선택지'다.
- **영속성이 강하다 — 서버가 죽어도 플레이 기록은 살아남았다** — 로그 §10-0 — vite 서버 사망 후 재시작·새로고침에도 dive 대화가 장면·arc·전체 턴까지 완전 복원됐다. '오늘 밤 한 게 내일 남는가'라는 내 최소 요건의 절반(기록 보존)은 통과다.

	- 컴포저 버튼(지금 응결/⏳계속/⏭전개/✦전개 후보)부터 설명 없는 한자어·아이콘 조합이고, PLAN에 들어가면 갓 만든 빈 프로젝트가 '하니스 37/100 · 보강 필요', '품질 게이트 강제 실패 5', '온톨로지 0개'로 시작한다. 아직 아무것도 안 한 사용자에게 빨간 실패 지표부터 보여주는 죄책감 UI다. 출간 화면의 '초안 없음 출간 준비' 헤드라인과 'release gate' 영한 혼용, 빈 프로젝트인데 게이트 2/3 ✓ 표시는 지표 자체의 신뢰까지 갉아먹는다. WRITE 하단의 무라벨 아이콘 6개도 같은 문제.
	- 근거 — 증거 팩 §6(컴포저 라벨, 툴팁 미확인), §10(PLAN 지표·용어 나열, '죄책감 UI' 관찰), §11('초안 없음 출간 준비', 빈 프로젝트 게이트 ✓), §9(무라벨 아이콘 6개 해독 불가).
	- 비교 — 카카오페이지·리디는 내부 QA 지표를 소비자 표면에 절대 노출하지 않고, 노션 같은 도구도 빈 문서를 '실패 5'로 맞이하지 않는다. 전문 용어는 기능을 쓰는 순간 문맥으로 가르치는 게 유료 제품의 온보딩이다.
- **[중간] 내보내기가 JSON뿐 — 내가 만든 이야기를 자랑할 수도, 들고 나갈 수도 없다**
	- ⋯메뉴가 출간/JSON 내보내기/JSON 가져오기 3개뿐이라 텍스트·EPUB·DOCX는커녕 본문 복사 동선조차 없다. 로판 독자의 과금 동력 중 하나는 '이 장면 봐'라며 커뮤니티에 공유하는 순환인데, 카일런의 위조 서신 장면을 트위터에 올릴 방법이 개발자용 JSON뿐이면 바이럴 루프가 원천 차단된다. 응결 실패 시 구제 경로 부재 문제와도 겹친다 — 텍스트 반출만 있었어도 차단 결함의 절반은 완화됐다.
	- 근거 — 증거 팩 §11 '내보내기가 JSON뿐 — 텍스트/EPUB/DOCX 없음', §8 반출 대안 부재 관찰.
	- 비교 — 리디·조아라 계열은 텍스트 반출이나 최소한 본문 선택·복사가 되고, 제타조차 대화 공유 이미지를 만들어준다.
### 있을 줄 알았는데 없던 것
- 계정·로그인과 기기 간 동기화 — 아이패드+폰 사용자를 위한 이어하기가 있을 줄 알았는데 localStorage뿐
- 서브남주 추가·상대 커스텀 수단 — 악역 영애물인데 로맨스 라인을 늘리거나 새 인물을 들일 입구가 어디에도 없음
- 응답 스트리밍 — 2026년의 대화형 AI 서비스에서 40초 무반응 대기는 있을 수 없는 일이라고 생각했다
- 응결 실패 시 플레이 기록 텍스트 반출·초안 수동 전환 같은 안전망
- 랜딩의 샘플 작품·데모 — '캐논이 30화 뒤까지 같은 얼굴'이라는 주장을 보여주는 실제 결과물이 하나도 없음
- 장면 공유 기능 — 잘 나온 대사를 이미지나 링크로 내보내 자랑할 동선

## 유료 시장 벤치마크 (2025-26 웹 조사)
유료 시장은 이미 세 축으로 성숙해 있다 — 캐릭터챗(제타·c.ai·크랙·Talkie)은 월 1만원 안팎 구독 또는 건당 35원 수준 마이크로 과금으로 '즉시 스트리밍 응답'을 기본기로 팔고, 작가 도구(Sudowrite·NovelAI·Novelcrafter)는 월 $4~59에 Story Bible·Lorebook·Codex라는 캐논 관리 상용 선례와 표준 포맷 내보내기를 갖췄으며, 플랫폼·변환(노벨피아·타입캐스트·AI Dungeon)은 독자 수익화와 오디오북 변환까지 실제 돈이 도는 사슬을 완성했다. Story X의 차별점인 '전개 후보 4개+의외도 태그'와 '캐논 일관성 게이트'는 이 10개 서비스 어디에도 없는 조합이지만, 실측 증거 팩이 보여주듯 턴당 35-45초 무스트리밍, 응결 120초 타임아웃 4연속 실패(실제 소요 141초), JSON 단독 내보내기, 요금·계정 부재라는 기본기 격차가 차별점 체감 이전에 이탈을 만든다. 즉 Story X는 '아무도 안 하는 것'(연속성 하네스)에서는 앞서 있으나 '모두가 하는 것'(응답 즉시성, 생성 신뢰성, 반출 경로, 과금 인프라)에서는 유료 최저가 티어($4~10)에도 못 미치는 위치다. 경쟁 우위를 지키려면 의외도 태그·캐논 게이트를 유지한 채, 스트리밍 도입과 응결 타임아웃 마진 수정(120→180초+ 또는 서버측 잡화), 텍스트/EPUB 반출부터 유료 시장의 바닥 기준선에 맞추는 순서가 합리적이다.
