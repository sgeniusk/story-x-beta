- **관찰(혼란)**: 상단에 도윤·하란·세하(무뚝뚝한 소꿉친구/떠돌이 검객/새벽 라디오 DJ) 칩이 악역 영애 설정과 무관하게 공존 — 용도 설명 없음.
- **관찰(코스메틱)**: 인디케이터에서 건너뛴 "작가 인터뷰"가 ✓ 완료 표시됨.

## 6. PLAY (dive) — 악역 영애에 빙의한 독자
- 상단: PLAY/WRITE/PLAN 토글. 파트너 요약(욕망·말투). 현재 장면 편집 가능한 텍스트박스 + 🪄쇼러너 버튼.
- 컴포저: 보내기 / 지금 응결 / ⏳계속 / ⏭전개 / ✦전개 후보 — 아이콘+한자어 라벨, 신규 사용자에게 설명 없음(툴팁 미확인).
- 턴1 전송 → "카일런 입력 중..." 표시. **스트리밍 없음, 경과 타이머 없음**, 응답 ~35–45초.
- 턴1 응답 품질: 나레이션(부채 끝에서 시선) + 위조 파혼 서신 훅 + 말투 준수("영애"), 🎯arc 질문 자동 생성("나는 누가 내 이름으로 파혼을 꾸미는지 밝혀내고... 처형될 운명을 피할 수 있을까?") + 선택지 칩 3개(조건부터 밝힌다/서신의 필체를 확인한다/전달자를 묻는다)
- **관찰**: 선택지 칩 클릭 = 즉시 전송(컴포저에 넣어 편집할 기회 없음).
- **관찰(자동화 특이)**: 일반 computer click이 칩에서 React onClick을 안 태우는 케이스 재현(dispatchEvent 필요) — 실사용자 무관, 단 레이아웃 시프트로 오클릭 유발 요인은 있음.
- 턴2(칩 "서신의 필체를 확인한다"): 필체 감식 디테일(문장 끝 꼬리 눌림=정서체 모사) + 자정 유예·공동 조사 제안 + 시험하는 눈빛 — 정합·긴장 유지. 응답 ~40초.

## 7. ✦ 전개 후보 (VS 주입 하네스)
- "전개 후보를 펼치는 중… 잠시만요." 후 ~30초 뒤 4개 후보, 각 후보에 **의외도 태그** (새 긴장 ×3 / 회수만 ×1).
- 후보 품질 매우 좋음 — 특히 4번: "편지의 문장이 원작에서 악역 영애가 처형 직전 남긴 유서와 같음을 깨닫고 위조임을 숨긴 채 동문 마차에 스스로 탄다" (메타 반전, 대가 명시 구조 '~하는 대가로 ~').
- 후보 선택 → 트랜스크립트에 "(전개 — …)" 회색 지시로 삽입 → 다음 턴이 그 방향으로 정합 전개(가시관 표식, "숨기는 것이 범인의 정체입니까, 영애 자신의 계획입니까?").
- **관찰(UX)**: 후보 4개가 각각 3줄 분량의 밀도 높은 문장 — 읽는 부담 큼. 라벨/요약 없이 전문 노출.

## 8. 지금 응결 — 실패 후 재시도
- 클릭 → "이야기를 한 회차로 응결하는 중… 수십 초 걸릴 수 있어요." (정직한 시간 힌트 있음)
- **버그(치명 후보)**: 1차 시도 `POST /api/dive-condense` → **net::ERR_ABORTED** → UI "응결 요청에 실패했습니다. 다시 시도하세요." 원인 정보 0. dive-chat 3건은 전부 200.
- **버그(시각)**: 응결 대기 중 뷰포트가 거대한 검은 공백(트랜스크립트 사라진 위치에 스크롤 고정, scrollTo(0,0) 해도 앱이 다시 바닥으로 강제 — 사용자는 검은 화면만 봄). 헤더도 fixed가 아니라 함께 밀림.
- 2차 시도도 **동일 실패** — 네트워크 로그상 두 건 모두 `net::ERR_ABORTED` = 클라이언트 120초 AbortController 타임아웃(diveClient.ts `DIVE_TIMEOUT_MS = 120000`, 주석 "응결(고급 모델)은 느려서 넉넉히 120초" — 실제 codex 응결이 오늘 120초를 초과). 3차 시도 진행.
- **관찰**: 응결·전개 후보·대화가 전부 수십 초급인데 스트리밍/진행률 없음 — "AI 대기" 감각이 누적.
- **관찰(구제 경로 부재)**: 응결 실패 시 플레이 기록을 초안으로 수동 반출/복사할 대안 없음. 40분 플레이가 트랜스크립트에만 존재.

## 9. WRITE (응결 실패 상태)
- warm 톤 에디터, "제목 없음" + "오늘 첫 문장을 써보세요" + 「초안 생성」 버튼. 0자.
- 헤더 작품명은 "악역 영애에 빙의한 독자"인데 에디터 제목은 "제목 없음" — 제목 이원화.
- PLAY에서 벌어진 사건이 WRITE에 전혀 안 보임(응결 전이라 의도된 동작이나, 실패가 반복되면 사용자에겐 "플레이는 헛수고"로 보임).
- 하단 툴바: 무라벨 아이콘 6개(문단/곡선/차트/펄스/인물0/전체화면) — 신규 사용자 해독 불가.

## 10-0. 응결 근인 확정 + 서버 사망 + 복원
- 응결 총 4회 시도 전부 실패. 1·2·4차 = `net::ERR_ABORTED`(클라 120초 타임아웃), 3차 클릭들 = 서버 사망으로 `ERR_CONNECTION_REFUSED`.
- **근인 실측**: 동일 트랜스크립트로 CLI 직접 실행 `node tools/storyx.mjs dive-condense --provider codex ...` → **성공, 2분 21초(141초)**. 출력 품질 훌륭(제목 "자정까지의 공범", hook "파혼을 꾸민 자는 ... 두 사람이 서로를 의심할 시간까지 계산해 두었다."). 즉 **응결 실패는 생성 실패가 아니라 120초 클라 타임아웃 < 실제 141초 라는 마진 설계 문제**.
- 응결 2회 타임아웃 직후 **vite dev 서버 프로세스 자체가 사망**(포트 5175 리스닝 소멸) — 재현 1회, 원인 미확정(브리지가 spawn한 codex 자식 프로세스는 클라 abort 후에도 계속 도는 구조).
- 서버 재시작+새로고침 후: 랜딩으로 착지(작업 중이던 프로젝트 자동 복귀 없음) → 프로젝트 목록에서 재진입 → **빈 WRITE로 착지**(PLAY 아님) → PLAY 탭 누르면 **dive 대화 완전 복원**(장면·arc·전체 턴 생존). 영속성 자체는 강함.
- 프로젝트 카드 제목 "악역 영애에 빙의한 독자 — 원작 전" — myRole 문장 중간 절단 그대로 노출.
- dive의 ← 뒤로 = 빈 WRITE로 이동(플레이 맥락에서 기대와 다름).

## 12. 함께 구상 (onboard-chat, S2 신기능) — 성공 경로
- "수다 떨듯 소재를 캐봅시다." 지하철 괴담 소재 2턴 주고받음 — 파트너가 재료를 되받아 **한 번에 질문 하나**(속삭임이었나 담담했나 → 찾아다니나 외면하나). 응답 ~15–20초.
- 「이걸로 시작」 강제 응결 ~15–20초 → 한 문장 reply + 시드 카드(첫 장면/내 역할/대화 상대 "이름 모를 승객") → 「이 설정으로 계속」 → playseed 확인 카드. **끝까지 매끄럽게 작동.**
- **동일 문제 재확인**: playseed 상단에 이번에도 무관한 도윤/하란/세하 칩 공존.
- 참고: DOM에 작가 인터뷰 캐러셀 패널 상존 — 인터뷰어 페르소나 "아가타 크리스 · 첫 장면에 수수께끼를 거는 플롯의 장인", 7문항, 첫 질문 "이번 작품은 독자에게 어떤 회차 경험을 약속해야 하나요?" (매 회차 후킹[추천]/느린 몰입/한 방의 반전/기타). 첫 회차 생성 화면 카피 "보통 2~3분쯤 걸려요. 새로고침하거나 창을 닫지 말고 기다려 주세요."
