## 종합 발견
1. **모든 매체가 매체 적합 콘텐츠 생성** — codex 가 매체에 적응(소설 서사·만화 시각감정·에세이 성찰·학술 초록/가설·오디오 분위기). 단위 명사도 분기(연재=회차, 단독=원고).
2. **인물 추출(P5/P6)이 전 매체 작동** — 만화 2인·에세이 화자·학술 연구자·오디오 2인 클린 승격(조사버그 0).
3. **★ 품질 게이트는 매체 특화** — 소설 8 · 만화 8 · 학술 **11**(A2~A4 주장-근거·인용·반론 추가) · 오디오 **7**. 매체별 게이트 셋이 다르다 = 매체 특화가 `qualityGates` 에 배선됨.
4. **★ 라이브 검토는 매체 무관(갭)** — floating 마진 검토는 매체와 무관하게 **고정 5인(쇼러너·캐릭터·세계·장르·연속성)** 만 돈다. blueprint `agentStack` 의 매체 특화 작가진(웹툰 연출·말풍선·다빈치·낭독 연출·논증 구조 등)은 **라이브 검토에 미배선.** 매체 특화가 게이트엔 있으나 검토엔 없다.
5. **온보딩 첫 회차 build codex transient 폴백** — #3 에서 "Reading additional input from stdin" raw 에러가 프로즈에 누수(폴백 초안). CLI 격리로 transient 확인. 개선 후보 — 폴백 시 raw 에러 가드 + 재시도(P1 묶음).
