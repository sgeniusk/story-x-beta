## ★ 사전 발견 — 실생성 작품에서 갈림길이 아예 안 뜨는 구조 갭 (수정 `a425042`)

분기점(1화 잠금) 상태에서 갈림길 카드가 **렌더되지 않았다**. 원인은 fork 도출 소스 2개가 실생성 경로에서 모두 빈다는 것.
- codex 가 `rewardArc` 의 payoff 를 **회차 안에서 즉시 채워** 미회수 promise 가 0.
- `openThreads` 는 생성 경로 어디서도 채워지지 않음.
- 정작 "미뤄진 위험"은 `stakesLedger` 의 `deferred` 3건(서가을 정신 안전·붕괴 위험·백도현 진실)에 있는데 fork 소스가 아니었다.

기존 #2 검증은 23화 레저를 **합성 주입**한 시나리오라 이 갭이 보이지 않았다. **수정** — stake 별 최종 결말이 deferred 인 항목을 'deferred-stake' fork 로 추가(TDD 2케이스, kept/lost 로 결판난 stake 는 제외). 수정 직후 갈림길 카드가 deferred 3건으로 정확히 렌더.

## 핵심 실증

1. **갈림길 = 페이오프 가속기.** A암에서 fork 시드("…더 미루지 않고 인물의 선택과 대가로 결판낸다")가 rewardArc 약속으로 직결돼 회차 내 이행. ch1 deferred 2건이 ch2 에서 kept 로 회전하고 새 deferred 가 다음 활주로를 엶 — **continuity≠payoff 문제(로판 최대 발견)의 구조적 해법이 작동**.
2. **B암 = 쇼러너 deferral 압박 재현.** B암 ch2 쇼러너가 "발견을 더 쌓기보다 대가로 전환하라" — 갈림길 없이는 생성이 발견 축적으로 흐른다는 통제군 증거.
3. **과회수 신호 — 회차 진도 인터뷰(follow-up a) 승격 근거.** A암 ch2 검토 4건이 수정 — "심문이 정답을 너무 빨리 건넴"·"고백이 관계 델타를 앞지름". 시드 문구가 일률적으로 "결판낸다"라 페이스 조절 입력이 없음. **권고 — (1) 시드 강도 2단(결판/진척) (2) 회차 진도 인터뷰를 1순위 follow-up 으로 승격.**
4. **P7 신규 — fork 시드가 의도 메모에 잔류.** 회차 잠금 후에도 이전 화 시드가 남아 다음 화에 이미 회수된 약속을 재지시함(이번엔 수동 정리로 회피). 잠금 시 소비된 시드 제거 또는 회차별 메모 스코프 필요.
5. **stake 문자열 드리프트.** ch1 "백도현의 정체와 게이트 조작의 진실" vs ch2 "백도현과 게이트 조작의 핵심 진실" — 문자열 키라 다른 stake 로 취급돼 회수된 위험이 fork 옵션에 잔존. P5/P6 계보의 stake 판 — 정규화 또는 유사도 매칭 후보.
6. **Q2 폴백 가드 라이브 검증.** A암 ch3 1차 시도가 (머지로 인한 vite 리로드로) 실패 → 결정론 폴백 초안 생성, **프로즈에 raw 에러 누수 0** (#3 온보딩의 "Reading additional input from stdin" 누수와 대조). 단 폴백 품질 자체는 템플릿 질감(조사 오류 "동료은"·장르 무관 소품) — 별도 개선 후보.
7. **P1 빈응답 1/13** (B암 ch3 연속성). 재시도 가드 머지 후에도 잔존 — 빈도는 낮아짐.
8. **(동시 검증) 2d floating 출간** — A/B 잠금 동선에서 FloatingPublishWorkspace 실데이터 렌더·잠금·복귀 정상. 캡처 `floating-publish-2d-live.png`.

## 결론

갈림길(작가 결정)은 **연속성(양 암 5점 동률)이 아니라 페이오프·상업성 축을 움직인다** — 정확히 설계 의도대로. 단 현재 시드는 "회수하라"만 말할 수 있고 "어떤 페이스로"는 말할 수 없다. 다음 조각은 (1) deferred-stake fork 안착(이번 수정), (2) 시드 강도 2단화, (3) **회차 진도 인터뷰**(승격), (4) P7 시드 잔류 정리.
