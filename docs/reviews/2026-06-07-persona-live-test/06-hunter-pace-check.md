# #3 헌터물 — 진도 체크 실효 관찰 (2026-06-10 3차, ch4~5)

- **목적** — 진도 체크 카드(paceInterview MVP)와 페이스 보강 시드가 A/B 에서 본 과회수를 실제로 줄이는지 전후 관찰. 기준선 = A암 ch2(과회수류 수정 4건: "정답을 너무 빨리 건넴"·"고백이 관계 델타 앞지름" 등, `05-hunter-ab-forks.md`).
- **방식** — A암(3화, deferred 1건 주입) 위에서 회차마다 진도 시드+갈림길을 걸고 ch4·ch5 생성, 각 5인 검토. 전후 비교(통제군 아님)임을 명시.

## 회차별 경과

| | ch4 "관측자의 문턱" | ch5 "고백의 비용" |
|---|---|---|
| 진도 시드 | 중턱 · 전진 · 다음 회수 1~2화 안 + fork(합류, 비정체 연성) | 정상 직전 · 회수 · 이번 화 + fork(숨긴 진실) |
| 생성 결과 | 합류 payoff 가 **단서 수준으로 절제**(표식·메모 발견). 새 약속 "숨긴 진실"을 **payoff 빈 채로 남김** — 이전 패턴(전부 즉시 회수)과 다름 | ch4 가 미룬 약속을 1화 뒤 고백으로 회수("1~2화 안" 이행). 합류는 계속 절제·이월. 새 hook(한지욱의 조건부 연락) |
| 검토 판정 | 수정3 · 결정1 · 관찰1 | 결정3 · 수정1 · 관찰1 |
| 과회수류 지적 | **0건** (수정 3건은 힐러 규칙 경계·해설 문장 — 정합성/문체 이슈) | **1건** (장르 — 중후반 한 호흡 밀도. 회수 회차 특성 감안) |

기준선 A암 ch2 의 과회수류 4건 → ch4 0건·ch5 1건. **진도 시드가 페이스 신호를 생성에 실제로 전달한다.** 쇼러너 ch5: "이번 회차의 약속을 제대로 회수한다 … 더 좁고 날카로운 선택으로 다음 회차의 압력을 만든다."

## 함께 라이브 검증된 것

- **P7 + intentVersion (`066ea9f`)** — ch4 생성 직후 strip 이 state 에만 반영되고 uncontrolled textarea DOM 에 시드가 잔류하는 갭 발견 → bodyVersion 패턴의 `intentVersion` 재시드 키 추가. ch5 생성 후 메모 4줄 → 0줄 자동 소거 확인.
- **진도 카드 사이클** — 질문 3 렌더·클릭 합성·같은 질문 교체·생성 후 소거가 2회차 연속 정상.

## ★ 신규 발견 P12 — 갈림길이 캐논과 모순된 약속을 걸러내지 못함

ch5 에서 연속성 감수자·캐릭터 큐레이터 **2인이 독립적으로 캐논 충돌 적발(출고 불가)** — "태준이 서가을에게 미래 진실을 고백"은 ch2 에서 이미 일어난 확정 캐논인데, ch4 생성 LLM 이 이를 "숨긴 진실을 말하는가?"라는 **새 미회수 약속으로 재생성**했고, fork 는 rewardArc 텍스트만 보고 옵션으로 노출, 작가(나)가 선택해 ch5 가 첫 고백처럼 다시 썼다.

- **검토 망은 설계대로 작동** — "연속성 검사를 약화시키지 않는다 — 충돌은 드러낸다" 원칙의 정확한 실증. 2인 수렴 + retcon note 요구까지.
- **갭은 상류에 있음** — (a) 생성 LLM 이 캐논 digest 에 있는 기확정 사실을 새 promise 로 재발급 (b) buildEpisodeForks 가 promise 의 캐논 정합성을 못 봄.
- **개선 후보** — fork 옵션 도출 시 promise 를 canonFacts 와 보수 매칭(stake 드리프트 매처 재사용 가능)해 기확정 사실과 겹치면 옵션에서 제외 또는 "캐논 확인 필요" 배지. 별도 스펙 권장.

## 결론

1. **진도 체크 MVP 실효 확인** — 과회수류 지적 4건→0~1건, 절제(ch4)→회수(ch5) 사이클이 시드 지시대로 작동. **2단계(쇼러너 서술형 인터뷰) 착수 가치 있음.**
2. 연속 생성은 ch5 캐논 충돌(출고 불가 판정)에서 중단 — 작가 행동으로도 수정 시점이며, P12 가 다음 우선 수정 대상.
3. 관찰 한계 — 전후 비교(같은 회차 아님)·표본 2회차. 2단계 설계 전 P12 수정 후 한 사이클 더 관찰 권장.
