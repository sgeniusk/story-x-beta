- **#23 (med)** 데이터 모드 기본 뷰가 매체 무관 인물 관계도 고정(`StoryXDesk.tsx:586`) → 에세이/학술은 인물 0이라 **빈 화면+추가 수단 없는 막다른 길**. medium별 기본 뷰 분기 필요(DataView category 설계).
- **#20 (low)** 확정 회차 수 입력이 min/max 미강제(`App.tsx:1374`, 0/5/999 수용) — onChange clamp 는 타이핑을 방해하므로 onBlur 방식이 적절.

## 3. 검토 대기 — 흐름 재설계·새 기능 (14건, 자동수정 안 함)

### 🔴 최우선 (데이터 손실·안전망)
1. **#1 (freq 8, 전 prose 매체) — 편집 본문 무음 소실.** `editorText`를 `chapter.prose`로 쓰는 핸들러가 코드에 0개. 회차 전환·새로고침·JSON 가져오기 시 본문이 사라지는데 상단은 내내 "저장됨" 오표시(하드코딩 문자열). → editorText→chapter.prose commit 경로(명시 저장 또는 전환/blur flush) + dirty 가드 + 정직한 저장 상태 + beforeunload 가드. **데이터 손실이라 1순위.**
2. **#1-undo (freq 10, 전원) — 바이블 편집 되돌리기 전무.** updateXxxMemory 가 스냅샷을 push 안 함. "3~6회 갈아엎기"의 안전망 부재. → per-edit 스냅샷 또는 logCanonChange.before 기반 revert.
3. **#3 (freq 7) — 설정 변경 영향 회차가 편집 지점에 안 뜸.** CanonRefactorPanel 이 canon 섹션에서만 렌더, 편집 입력 지점(CanonCanvas)엔 없음. 랜딩/인터뷰 "영향 범위 먼저 펼쳐 보여줍니다" 약속 정면 위반.

### 🟠 중간 (수정 루프 골격)
4. **#4-picker (freq 5) — FloatingEditor 회차 선택기 부재** + 단편(short-novel)은 `isSerial=false`라 picker 게이트 차단 → 과거 회차 본문 편집 동선 없음.
5. **#5 (freq 4) — 잠긴 회차도 `editable=true`** — 경고·읽기전용 없이 타이핑되고 저장도 안 됨. `unlockChapter`는 정의만 있고 호출처 0(잠금 해제 UI 없음).
6. **#6 (freq 6) — 캐릭터/캐논 CRUD 부재.** 추가·삭제·이름/역할/관계 변경 핸들러 0개. 욕망·상처·현재상태 3개 textarea 덮어쓰기만 가능.
7. **#7 (freq 4) — 잠근 헌장(결말·4줄·화수) 재열람·수정·증보 UI 전무.** `ContractAmendment`·`validateContract` 모델은 있으나 미배선. 생성은 계속 옛 척추로 발화.
8. **#10 (freq 3) — 매체/형식 변경이 경고·confirm·영향분석 없이 즉시 덮어쓰기**(기존 회차·작가진·헌장 무음 전환).
9. **#17 (freq 3) — 떡밥·회차 구조 비트·에피소드 보드가 읽기 전용**(추가·편집·재배치 affordance 0).

### 🟡 매체 특화 가치 (제품 차별성)
10. **#8 (학술) — claim/APA citation/integrity 마진검토가 dead code.** `displayedMarginReviews` 소비처 0개라 학술 제품 가치 자체가 화면에 안 보임. → FloatingEditor 에 전달.
