# Story X 서비스 런칭 계획

> 작성 2026-05-16 · 기준 버전 `Alpha v0.8.0`.
> **이 문서가 Story X 계획의 단일 출처다.** 과거 문서(`story-x-prd.md`, `agent-workflow-plan.md`, `storyx-evaluator-development-update.md`)는 배경 자료로만 둔다. 버전별 변경 기록은 `storyx-version-log.md`.

## 0. 한 줄 정의

Story X는 창작자가 만든 하나의 이야기가 형태(소설·에세이·만화·오디오북·영상)를 바꿔도 영혼을 잃지 않게 돕는, 이야기 우선 창작 운영체제다.

## 1. "런칭"이란 무엇인가

런칭은 기능을 다 만든 시점이 아니라 **외부 창작자가 자기 계정으로 들어와, 결제 없이도 첫 작품을 끝까지 만들고, 유료로 올라갈 이유를 스스로 느끼는 시점**이다.

런칭이 성립하려면 아래가 모두 참이어야 한다. 이것이 런칭 게이트다.

1. 창작자가 로그인해 만든 작품이 다음 접속에도 그대로 남아 있다.
2. 초안 생성과 작가진 검토가 그 창작자의 환경에서 (개발자 로컬이 아니라) 동작한다.
3. 장편 10화 이상에서도 캐릭터·세계·캐논이 무너지지 않는다.
4. 사용자가 직접 고친 문장을 AI가 무단으로 덮지 않는다.
5. 실패해도 비용이 새지 않고 중간 결과가 보존된다.
6. 무료 가치와 유료 가치의 경계가 분명하다.
7. 약관·개인정보·저작권 귀속이 정리돼 있다.

## 2. 지금 상태 — v0.8.0

**되는 것**
- 랜딩 → 로그인 → 프로젝트 → 새 프로젝트 → 에디터 흐름, 편집/바이블 2트랙 에디터.
- 초안 생성·작가진 검토가 실제 Claude 호출로 동작(로컬 dev 브리지 경유).
- 2화 이상 생성 시 확정 캐논·인물·세계를 LLM에 넘겨 연속성 유지.
- 메모리 승인 큐, 캐논/캐릭터/세계/문체 바이블, ⌘K 명령 팔레트, `storyx` CLI 하네스.

**아직 아닌 것 (런칭까지의 거리)**
- 검토에서 승인한 기억이 실제 작품 캐논으로 반영되는 루프가 닫히지 않았다.
- LLM 생성이 개발자 로컬 브리지에서만 동작한다. 외부 사용자 환경에서는 동작하지 않는다.
- 계정·프로젝트 영속 저장·클라우드가 없다. 새로고침하면 작품이 사라질 수 있다.
- 장편 회귀 검증, 한국어 문체 유지 평가가 없다.
- 만화 컷 편집기·오디오 라인 편집기는 초기 설계 단계다.
- 결제·크레딧·약관·개인정보 정책이 없다.

## 3. 런칭까지의 단계

각 단계는 "완료 기준"을 통과해야 다음으로 넘어간다.

### 단계 A · 핵심 창작 루프 닫기 — `v0.9.0`

검토가 만든 기억 후보가 승인을 거쳐 실제 작품 상태로 들어가게 한다. 이게 닫혀야 생성·검토·연속성이 한 루프가 된다.

- 검토 → `memoryCandidates` → 승인 대기함 → 사용자 승인 → `project`의 canon/character/world 갱신.
- 갱신된 캐논이 다음 회차 생성 컨텍스트에 자동 반영.
- 직접 편집 diff를 변경 블록 단위로 표시하고, 검토 패킷에는 diff만 포함.
- `review-ledger`와 실패 로그 기록.

완료 기준 — 회차를 생성·검토하고 후보를 승인하면 바이블에 반영되고, 다음 생성이 그 캐논을 지킨다.

### 단계 B · 사용자가 쓸 수 있는 LLM 운영 — `v0.10.0`

지금 LLM 생성은 개발자 로컬에서만 된다. 서비스가 되려면 외부 사용자 환경에서 동작해야 한다. 이 단계의 핵심 결정이다.

- 생성·검토를 서버 측에서 실행하는 경로 구축(Vercel Function + Anthropic API).
- 비용·실패·재시도·타임아웃 처리. 실패 시 사용자에게 비용을 물리지 않는다.
- (선택) 사용자가 자기 API 키 또는 구독 토큰을 연결하는 BYO 경로.
- 로컬 CLI(`storyx`)는 개발·고급 사용자용으로 계속 유지.

완료 기준 — claude CLI가 없는 외부 환경에서도 초안 생성과 검토가 끝까지 동작한다.

### 단계 C · 계정과 영속 저장 — `v0.11.0`

- 계정·로그인, 프로젝트가 서버에 저장되고 다음 접속에 복원된다.
- `storyx init`/`serve`/`memory sync`로 로컬 프로젝트 폴더와 웹이 같은 상태를 공유.
- 프로젝트 버전 히스토리와 복원.

완료 기준 — 창작자가 작품을 만들고 닫았다가 다시 열어도 캐논·회차·승인 상태가 그대로다.

### 단계 D · 첫 완성 알파 — `v1.0.0-alpha`

아이디어 → 작가 인터뷰 → 초안 생성 → 직접 편집 → 흐름 검증 → 바이블 승인 → 출간 잠금. 이 루프가 소설과 에세이에서 끊기지 않는다.

완료 기준 — 장편 10화 이상 생성·수정해도 캐릭터 욕망·세계 규칙·사건 타임라인이 유지되고, 에세이에서 AI가 없는 기억을 지어내지 않는다.

### 단계 E · 비공개 베타

- 실제 창작자 10명 이상이 작품을 끝까지 제작(단편·장편·에세이 표본).
- 재미·일관성·직접 편집성·검토 신뢰도를 측정하고 실패 로그를 모은다.
- 만화 컷 보드·오디오 낭독 보드 등 매체별 제작을 이 단계에서 실사용 검증.

완료 기준 — "AI가 내 글을 빼앗았다"는 순간이 거의 없고, 검토 후 수정률·만족도가 기록된다.

### 단계 F · 공개 베타와 과금, 그리고 정식 런칭

- 크레딧·검토 규모별 사용량 표시, 무료 생성 / 유료 깊은 검토·매체 전환.
- 약관·개인정보(특히 에세이 민감정보)·저작권 귀속 정리.
- 플랫폼별 출판·내보내기 패키지.
- 정식 런칭 — 클라우드 저장, 팀 협업, 장기 연재 메모리 버전 관리.

완료 기준 — 1조 런칭 게이트 7개 항목이 모두 참이다.

## 4. 우선순위

- **지금 당장(P0)** — 단계 A(승인 루프 닫기). 이게 막혀 있으면 생성·검토가 따로 논다.
- **그다음(P0)** — 단계 B(서버 측 LLM 운영). 외부 사용자가 없으면 서비스가 아니다.
- **P1** — 계정·영속 저장, 장편 10화 회귀 하네스, 직접 편집 diff.
- **P2** — 매체별 제작(만화·오디오), 매체 전환 브릿지, 결제·크레딧, 클라우드·팀 협업.

## 5. 제품 원칙 (불변)

- 이야기가 먼저이고, 매체는 그 다음이다.
- 원고 전체를 매번 컨텍스트로 밀어 넣지 않고, 메모리 뱅크에서 역할별 컨텍스트 패킷을 만든다.
- 캐논 충돌은 다수결로 통과시키지 않는다. reveal / revision / blocked 중 하나로 분류한다.
- 사용자가 직접 고친 문장은 AI가 덮어쓰지 않는다. 검토·대안·영향 범위를 먼저 제시한다.
- 새 기억과 새 캐논은 사용자 승인 뒤에만 반영한다.

## 6. 리스크

- **LLM 비용 구조** — 서버 측 생성은 토큰 비용이 발생한다. 무료 한도와 유료 경계를 단계 B~F에서 같이 설계해야 적자 런칭을 피한다.
- **장편 연속성** — 회차가 쌓일수록 컨텍스트가 커진다. 메모리 뱅크 요약 품질이 장편 성패를 가른다.
- **개인정보** — 에세이 매체는 실제 인물·민감 기억을 다룬다. 익명화와 보관 정책이 런칭 전 필수다.
- **창작자 신뢰** — AI가 사용자의 글을 덮어쓰는 경험이 한 번이라도 나오면 이탈한다. 직접 편집 보호가 최우선 품질 기준이다.
