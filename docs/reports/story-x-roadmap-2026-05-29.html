02

성공 요소 — 왜 다른가

생성형 글쓰기 도구가 풀지 못한 문제를 정조준한다

① 연속성을 편집 마무리가 아니라 제품 요건으로

대부분의 AI 글쓰기 도구는 한 번에 잘 쓰는 데 집중한다. Story X는 수백 회차에 걸쳐 캐릭터·세계관·캐논·복선이 무너지지 않는지를 시스템이 강제한다. 충돌은 숨기지 않고 드러낸다.

② 이중 하네스 — 코드 하네스 + 스토리 하네스

코드 하네스(progress·feature_list·init·handoff)가 개발의 연속성을, 스토리 하네스(온톨로지·연속성 계약·품질 게이트)가 작품의 연속성을 보장한다. 같은 철학을 제작과 산출물 양쪽에 적용한다.

③ 36명 검증 에이전트 + 16 craft 기준

쇼러너·캐릭터 큐레이터·연속성 감수자 등 역할이 분리된 작가진이 서로의 의견을 보지 않고 독립 검토한다. 마진(마지널리아) 모델로 의견이 본문 단락 옆에 anchored되어 길어져도 깔끔하다.

④ 하나의 작품 그래프, 5매체 투영

같은 StoryOntology가 소설·에세이·웹툰·인스타툰·4컷으로 투영된다. 매체별 필드는 달라도 핵심 4요소(극적 질문·욕망·비용·플롯)는 변하지 않는다.

지향점 — "한 작가가 30회차 분량 웹소설을
끊기지 않는 연속성으로 완성한다."
v1.0 출시 게이트의 핵심 정의 · 마스터플랜 x-zippy-graham
03

런칭 계획 — v1.0 까지

alpha 셀프체크에서 실사용 완주까지 4단계
L1

실 LLM end-to-end 연결 ~2주

현재 다수 경로가 mock 폴백이다. Vercel 환경변수(AI Gateway / Anthropic) 등록 후 생성·검토·인터뷰 전 루프를 실제 모델로 검증한다.

  • 배포본 5라우트 curl 검증
  • agentRunEngine generic 출력 → LLM 호출 전환
L2

영속성 + 인증 + 다중 작품 ~3주

작품 데이터가 localStorage에 갇혀 있다. 파일/클라우드 영속, 로그인, 작가 한 명이 여러 작품을 관리하는 라이브러리로 확장한다.

  • storyx CLI 확장 (init·serve·memory sync)
  • evolutionMemory 클라우드 싱크
L3

30회차 연속성 완주 게이트 ~3주

출시의 정의 그 자체다. 실제로 한 작품을 30회차까지 끌고 가며 캐논·복선·관계 온도가 무너지지 않는지 alpha 셀프체크를 100%로 끌어올린다.

  • 긴 호흡 연속성 회귀 테스트
  • 미해결 복선 부하·페이오프 스케줄 검증
L4

퍼블릭 베타 런칭 ~2주

제한 인원 베타로 실사용 피드백을 수집한다. 온보딩 활성화, 첫 작품 생성까지의 시간 단축, 데모 작품 동봉.

  • 온보딩 플로우 + 첫 회차 생성 데모
  • 사용 로그·실패 로그 기반 개선 루프
04

확장 계획 — v1.0 이후

매체·수익화·협업 세 축으로 넓힌다
영역내용시점
매체 풀 확장오디오북·교양영상·만화 제작 파이프라인 정식화. 현재 audiobook 매체가 투영 target에 미정의 — 매체 풀 보강.단기
수익화·과금크레딧·유료 검토 티어·전환 지점 설계. 토큰 소모(quick/standard/deep)와 연계한 가격 모델.단기
퍼블리시 자동화Kindle·리디·네이버·카카오 등 플랫폼별 포맷 변환과 메타데이터·발신 카피 자동 생성.중기
협업·공유공동 작업, 작품 바이블 공유, 편집자·작가 역할 분리. 작가실(writer's room) 다인 모드.중기
메모리 진화 고도화작품 수명에 걸친 학습 원칙 압축·드리프트 감지. 작가별 문체·선호의 누적 학습.장기
플랫폼화외부 작가·스튜디오가 자신의 페르소나·바이블·하네스를 얹는 멀티테넌트 확장.장기
📚

매체 — 한 작품, 여러 출구

하나의 작품 그래프에서 소설·웹툰·오디오북이 동시에 자란다. 매체 변환을 후처리가 아니라 투영으로 본다.

💳

수익 — 가치를 가두지 않는 과금

검토 규모별 크레딧과 유료 정밀 검토. 작가를 종속시키지 않으면서 지속 가능한 모델을 설계한다.

🤝

협업 — 작가실의 확장

AI 작가진 옆에 사람 협업자를 둔다. 바이블·캐논을 공유 자산으로 운영한다.

05

이번 마일스톤 — M10 디자인 통합

검수받은 외주 디자인을 코드에 반영, 분업으로 처리

분업 모델 — Codex 구현 · Claude 하네스/총괄

Claude가 검증 가능한 Task Packet(스코프·함정·게이트)을 작성하고, Codex(gpt-5.5 @ xhigh)가 실제 코딩을 맡았다. Claude는 게이트 3종(tsc·test·build)을 직접 재실행하고 라이브 렌더와 스코프 위반을 점검한 뒤 커밋했다. 도메인 로직은 단 한 줄도 바뀌지 않았고, 어댑터로 결과만 변환했다.

📝

우레일 — 마지널리아

길게 쌓이던 검토 카드를 본문 단락 옆 마진 의견(3단계 severity)으로. 코어 5명 슬리버 + ⌘K 호출.

🗂️

좌레일 — 구조 중심

평면 회차 목록을 기승전결 구조 트리 + "구조↔지표" 세그먼트로. 지표 탭에 가벼운 DataPanel 4카드.
