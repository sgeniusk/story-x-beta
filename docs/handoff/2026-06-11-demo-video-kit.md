# 완성 루프 데모 영상 제작 키트 (M7 경량 검증 — 방법 C)

> 목표 — 5분 이내 데모 영상(Loom/YouTube 비공개)으로 외부 작가 모집의 첫 소개 자료를 만든다.
> 성공 지표 — 시청자 5명 중 3명 이상 "써보고 싶다" (결정 문서 `docs/decisions/2026-06-10-market-proof-1.0.md` 방법 C).
> 이 키트는 녹화 전 세팅·장면별 콘티·나레이션 스크립트를 제공한다. 녹화는 사용자가 Loom 등으로 수행한다.

## 사전 세팅 (2026-06-11 라이브 검증 완료)

1. `npm run dev` (provider 는 codex 기본 — 라이브 생성 장면용).
2. 데모 상태는 백업 JSON 을 localStorage 로 주입해 재현한다. **백업이 두 형식이라 아래 스니펫이 둘 다 처리한다** — dev 서버에서 브라우저 콘솔에 붙여넣고 실행하면 새로고침까지 끝난다.

```js
// 파일명만 바꿔 실행 (dev 서버의 /@fs/ 절대경로 서빙 사용)
(async () => {
  const file = '02-work-backup-ch23.json'; // 또는 03-hunter-llmpace-ch6.json
  const r = await fetch('/@fs/Users/taewookkim/dev/story-x/docs/reviews/2026-06-07-persona-live-test/backups/' + file);
  let j = await r.json();
  if (typeof j === 'string') j = JSON.parse(j); // 형식 2 — 이중 인코딩(localStorage 키 맵)
  if (j.dump) { // 형식 1 — {_meta, dump:{project,snapshots,evolutionHistory}}
    localStorage.setItem('serial-story-studio/project', j.dump.project);
    if (j.dump.snapshots) localStorage.setItem('serial-story-studio/snapshots', j.dump.snapshots);
    if (j.dump.evolutionHistory) localStorage.setItem('serial-story-studio/evolution-history', j.dump.evolutionHistory);
  } else {
    for (const [k, v] of Object.entries(j)) if (k.startsWith('serial-story-studio/') || k.startsWith('storyx.')) localStorage.setItem(k, v);
  }
  location.reload();
})();
```

3. 주입 후 진입 동선 — 랜딩 "창작 시작" → 상단 "에디터로" (저장된 작품이 그대로 열린다).

| 장면 | 백업 파일 | 형식 | 재현 상태 (검증됨) |
|---|---|---|---|
| 완권·캐논·검토 | `02-work-backup-ch23.json` | 1 | 로판 23화 완결, 캐논 91, 온톨로지 113, 인물 4 |
| 갈림길·진도 인터뷰 | `03-hunter-llmpace-ch6.json` | 2 | 헌터물 6화 "감시를 켜는 밤", 갈림길+진도 카드 |
| 캐논 충돌 사고 재현(선택) | `03-hunter-pacecheck-ch5.json` | 2 | "출고 불가" 판정이 나온 5화 상태 |

4. 화면 — 1280×800 이상, 다크 캔버스(기본). 녹화 중 생성 대기(40~90초)는 편집에서 점프컷.
5. **장면별 UI 위치** (floating 편집기 하단 독 기준)
   - 갈림길 + 진도 체크 카드 — 독 **"상태"** 패널 안(작품 상태 → 이번 화 갈림길 → 회차 진도 체크·쇼러너에게 묻기).
   - 5인 검토 — 독 **"작가실"** 패널("5명에게 전체 검토 맡기기" 버튼).
   - 캐논 보드·인물 그래프 — 상단 **"데이터"** 모드 → 독 "캐논" → "인물4" 파고들기.
   - 회차 목록·체크리스트 — 상단 **"출간"**.
6. ⚠️ ch23 백업의 인물 중 "레오르 벨로트라"는 당시 추출 버그(조사 잔류)가 남은 옛 데이터다(코드는 이후 수정됨). 영상에서 인물 그래프를 확대할 땐 이 노드는 피하거나, 자막으로 "초기 버그, 수정됨"을 명시한다.

## 콘티 — 5분 (장면 7개)

### S1 · 0:00–0:25 — 훅 (문제 제기)
- **화면** — 23화 완권 작품의 회차 목록(출간 모드) 천천히 스크롤.
- **나레이션** — "AI 로 장편을 써 본 사람은 압니다. 10화쯤 가면 인물 이름이 바뀌고, 설정이 무너지고, 떡밥은 영영 회수되지 않죠. 이 작품은 AI 가 쓴 23화 완결 장편입니다. 확정 설정이 한 번도 무너지지 않았습니다."

### S2 · 0:25–1:10 — 시작은 세 문장 (온보딩→1화)
- **화면** — 새 작품 온보딩. freewrite 입력("몰락한 백작가의 막내딸로 빙의했다…") → 인터뷰 질문 → 1화 생성(점프컷) → 떠 있는 작업실(floating editor)에 1화 본문.
- **나레이션** — "시작은 세 문장입니다. 시스템이 인터뷰로 전제를 캐내고, 첫 화를 만들면서 로그라인과 독자 약속을 자동으로 심습니다. 여기서부터 모든 사실이 '캐논'으로 적립됩니다."

### S3 · 1:10–2:10 — 5인 검토진 (차별점 1)
- **화면** — 전체 검토 실행 → 마진에 5인 의견 도착(점프컷) → 의견 카드 확대(쇼러너·세계 키퍼·연속성 감수자).
- **나레이션** — "회차마다 다섯 명의 AI 검토진이 본문을 실제로 읽습니다. 쇼러너는 연재 압력을, 세계 키퍼는 행동의 비용을, 연속성 감수자는 캐논 충돌을 봅니다. 이들은 같은 원칙을 매 회차 반복 적용합니다 — 한 검토자는 '비용이 설명되지 않았다'는 지적을 9회 연속으로 던졌고, 본문이 그 압력을 수용하자 지적을 멈췄습니다."
- **하이라이트(가능하면)** — "출고 불가" 판정 화면. "검토진은 캐논과 모순된 회차에 '출고 불가'까지 선언합니다. 우리는 이 충돌을 숨기지 않습니다."

### S4 · 2:10–3:00 — 캐논 보드 (차별점 2)
- **화면** — 데이터 모드. 캐논 카드 그리드 스크롤(91개) → 인물 그래프(리아나·레나·루시안·관계 엣지) → 타임라인.
- **나레이션** — "확정된 사실은 캐논 보드에 쌓이고, 다음 화 생성에 강제로 주입됩니다. 초반에 회차마다 바뀌던 둘째 오빠 이름이, 캐논에 들어간 순간부터 23화까지 고정됐습니다. 캐논은 보존을 넘어 추리 엔진이 됩니다 — 고정된 이름과 날짜가 미스터리의 용의자 소거 논리로 작동했습니다."

### S5 · 3:00–4:00 — 작가의 통제 (갈림길 + 진도 인터뷰)
- **화면** — 헌터물 백업으로 전환(점프컷). 갈림길 카드(미뤄진 위험 → 옵션 클릭 → 의도 메모 합성) → "쇼러너에게 묻기" → 작품 맞춤 질문 3개 → 옵션 선택.
- **나레이션** — "AI 연재의 진짜 난제는 '틀리지 않게'가 아니라 '끝나게' 만드는 겁니다. AI 는 떡밥 회수를 무한히 미루는 경향이 있거든요. Story X 는 미회수 약속에서 결정 카드를 만들어 작가에게 묻습니다 — 이번 화에 결판낼까요, 한 호흡 쉴까요. 쇼러너에게 물으면 지금 작품 상황을 읽고 맞춤 질문을 던집니다. 작가가 고른 연출 지시는 다음 화에 그대로 이행됩니다."

### S6 · 4:00–4:40 — 완결 (페이오프)
- **화면** — 로판 백업 복귀. 23화 "처형 예정자의 첫 아침" 본문 + 1화 제목과 나란히(수미상관) → 캐논 카드 "운명 전환 확정".
- **나레이션** — "이 작품은 1화와 같은 제목으로 끝납니다. 1화부터 쌓인 떡밥 전부가 위반 없이 회수됐고, 주인공의 운명 전환이 캐논으로 확정됐습니다. '죽어야 끝나는 이야기에서, 살아서 책임지는 이야기로.'"

### S7 · 4:40–5:00 — CTA
- **화면** — 공개 패키지(`docs/public/`) 화면 또는 랜딩.
- **나레이션** — "전체 실증 로그를 공개해 두었습니다. 연재 경험이 있는 작가분들의 베타 참여를 기다립니다."

## 녹화 팁

- 생성·검토 대기는 전부 점프컷. 실시간으로 보여줄 것은 "클릭→결과" 인과뿐이다.

- 나레이션은 위 스크립트 기준 ~650자/분 — 장면당 1~2회 끊어 녹음하면 편집이 쉽다.
