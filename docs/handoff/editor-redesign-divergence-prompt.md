# Story X 에디터 — 3방향 발산 프롬프트 (1단계)

> [editor-redesign-brief.md](editor-redesign-brief.md) 와 짝. 외부 AI 디자인 도구(claude.ai · v0 · Lovable · Figma Make)에서 **에디터 편집 모드 1화면**의 서로 다른 시각 방향 2~3개를 받기 위한 래퍼 프롬프트. 방향을 고른 뒤 2단계(나머지 화면·반응형·데이터 모드 확장)로 넘어간다.

## 쓰는 순서

1. 도구에 **이 의뢰서 파일 + 깨짐 스크린샷 3장**(`editor-1440.png` · `editor-768.png` · `editor-360.png`)을 첨부한다.
2. 아래 프롬프트를 입력한다.
3. 3방향의 컨셉 한 줄 + 각 시안(HTML)을 받는다.
4. 마음에 드는 방향을 골라 "그 방향으로 360 반응형까지" 추가로 요청한다.
5. 고른 방향의 **코드 또는 스크린샷**을 Claude Code(나)에게 가져와 실제 코드에 이식·검증한다.

## 도구별 메모

- **claude.ai 웹** — 한국어·이미지 가장 매끄럽다. 한 메시지로 Artifact 3개를 순서대로 만들어 준다. 길이 제한에 걸리면 "다음 방향"으로 이어 받는다.
- **v0 (vercel)** — React + Tailwind 코드가 나와 우리 스택 이식이 쉽다. 한 번에 셋보다 **방향 A부터 3번 New Chat**이 품질이 안정적이다.
- 공통 — 파일 업로드가 안 되는 도구면 의뢰서 본문을 프롬프트 아래에 통째로 붙인다.

## 복붙 프롬프트

```
너는 시니어 프로덕트 디자이너다. 첨부한 브리프(Story X 에디터 재디자인 의뢰서)와
스크린샷(현재 깨진 상태 — 반례)을 읽고, 에디터 "편집 모드" 화면 하나를
제로베이스로 다시 디자인한 "서로 뚜렷이 다른 3가지 방향"을 제안해줘.

각 방향마다 다음을 줘.
1) 컨셉 한 줄 — 이 방향이 무엇을 가장 우선하는가
2) 1440px 기준 클릭 가능한 단일 HTML 파일 시안 (인라인 <style> + 바닐라 JS로
   모드 전환·집중 모드 같은 mock 동작)
3) 360px(모바일)에서 보조 패널이 어떻게 접히는지 — 짧은 설명 또는 360 시안

3가지 방향은 서로 겹치지 않게 축을 다르게 잡아줘 (더 좋은 축이 있으면 바꿔도 됨).
· 방향 A — 원고 절대 우선. 극단적 미니멀. 회차 구조·검토는 평소 숨겼다가 호출식으로 등장
· 방향 B — "작가실 보드". AI 작가진 5명의 존재감을 살린, 검토 카드가 중심 조연
· 방향 C — 집중 캔버스 + 플로팅 도구. 본문은 넓은 캔버스, 보조 도구는 떠다니는 패널/독

반드시 지킬 것.
· 다크 우선, 차분한 editorial 톤 (Scrivener·iA Writer 결).
  네온 보라/파랑 그라데이션·챗봇 말풍선·일반 SaaS 대시보드 금지
· 원고(본문)가 화면의 주인공
· 한국어 본문 (줄바꿈은 단어 단위), 본문 한 줄 45~75자
· 360·768·1024·1440 어디서도 안 깨짐 (상단바 회색 박스·세로로 쪼개진 글자·화면 밖 이탈 없음)
· 더미 데이터는 브리프 부록 A의 "샘플 작품"을 채워 넣을 것

먼저 3방향의 컨셉 한 줄씩을 요약해 보여주고, 그다음 각 방향의 시안을 순서대로 만들어줘.
(브리프 파일을 첨부하지 못했으면, 이 줄 아래에 의뢰서 본문이 이어진다.)
```

## 받은 뒤 — Claude Code(나)에게 가져오는 법

셋 중 편한 형태면 된다. 위로 갈수록 내가 정확히 이식한다.

1. **고른 방향의 HTML/코드 전체를 대화에 붙여넣기** (가장 정확)
2. **시안 스크린샷** + (가능하면) 코드 일부
3. v0/claude **공유 링크** — 접근이 막히면 코드/이미지를 대신 달라고 하겠다

가져오면 — 그 방향을 우리 `React + src/styles.css` 구조로 이식하고, `?stage=editor`를 360·768·1024·1440에서 재캡처해 안 깨지는지 증명한 뒤, 대화하며 조정한다.
