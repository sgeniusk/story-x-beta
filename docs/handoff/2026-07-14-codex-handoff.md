# Codex 이관 패킷 — Story X 주 구현 에이전트 인수인계 (2026-07-14)

Story X 구현 작업을 Codex 세션으로 이관하기 위한 패킷이다. 이 문서 하나로 Codex가 컨텍스트를 재구성하고 첫 슬라이스에 착수할 수 있어야 한다. 맨 아래에 복붙용 킥오프 프롬프트가 있다.

## 0. 전제

- **정본 지시 문서는 `AGENTS.md`다.** Codex는 `.claude/agents/*.md`를 네이티브 서브에이전트로 로드하지 못한다 — 같은 역할을 한 스레드 안의 개념적 전문가로 운용한다(AGENTS.md "Codex Compatibility" 절).
- 이 레포에는 이름이 같은 두 시스템이 있다 — **코드 하네스**(CLAUDE.md·progress.md·feature_list.json·init.sh·session-handoff.md, 코딩 에이전트용)와 **스토리 하네스**(docs/storyx-harness-architecture.md, 도메인 시스템). 혼동 금지.
- dev 서버의 LLM 호출은 vite 브리지가 로컬 `codex` CLI로 라우팅한다(vite.config.ts). 즉 Codex 세션이 코드를 고치고, 앱은 codex CLI로 스토리를 생성한다 — 두 층을 혼동하지 말 것.

## 1. 진입 순서 (매 세션)

1. `progress.md` — 현재 활성 작업·직전 검증 결과.
2. `feature_list.json` — 마일스톤 상태(`active: M11-review-driven-hardening`).
3. `session-handoff.md` — 맨 위 최신 인계(손대지 말 것 목록 포함).
4. `bash init.sh` — 변경 전 녹색 확인. **완료 주장 전에 한 번 더.**
5. `AGENTS.md` + `CLAUDE.md` — 운영·도메인 규칙.

## 4. 제안 백로그 (우선순위는 사용자 확정 사항)

아래는 검토 결과에서 도출한 권고 순서다. **어느 슬라이스부터 갈지는 사용자가 정한다** — 착수 전에 반드시 확인하라. 각 슬라이스는 이 레포 관례대로 brainstorm→spec→plan→TDD로 진행한다.

- **P0-a 응결 신뢰성** — 단순 타임아웃 연장(120→180초+)은 응급처치일 뿐이다. brainstorm 후보는 (1) 서버측 완주+클라 폴링(브리지가 잡을 끝까지 돌리고 결과를 회수 — 141초에 성공한 결과가 버려지는 현 구조 해소) (2) 스트리밍(SSE) (3) 타임아웃 연장+마진 계측. 어떤 안이든 `storyxBridge`의 abort→자식 kill 전파(발견 3)를 같이 다뤄라.
- **P0-b 실패 구제 경로** — 응결 실패 토스트 자리에 "플레이 기록 텍스트로 내려받기/초안으로 보내기". 작게 끝나는 독립 슬라이스.
- **P1-a 진행 피드백** — 온보딩 인터뷰 화면의 기존 관례(경과 타이머+단계 메시지, `generationProgress`)를 dive 대화·전개 후보·응결에 이식. 스트리밍 전이라도 체감을 바꾼다. 응결 대기 중 검은 공백/스크롤 강제 버그(발견 4)도 여기서.
- **P1-b 텍스트 반출** — 회차 본문 클립보드 복사 + TXT 내보내기. ⋯ 메뉴/WRITE에 배치.
- **P2** — 시드 칩 정리(발견 7), 내부 용어·죄책감 UI(발견 8), 칩 컴포저 삽입 옵션·인디케이터 ✓·제목 절단(발견 9), 응결 분량 파라미터(발견 6), prod dive-* Function(발견 10).
- **별도 트랙(기존 인계 유지)** — S3 적응형 인터뷰는 session-handoff.md 2026-07-13 노트의 설계 입력을 따른다. 신뢰성 백로그와 어느 쪽을 먼저 갈지도 사용자 결정.

## 5. Codex 필수 준수 조항 (위반 이력 기반 — 매 세션 적용)

과거 Codex 위임에서 실제 발생한 사고 패턴이다. 전부 재발 금지 조항으로 명시한다.

1. **동기 실행·완료까지 대기.** "백그라운드로 시작했다" 보고는 무효다 — 실제 파일 변경 없이 종료한 이력(no-op)이 있다.
2. **테스트 통과용 우회 주석 금지**(false-green). 게이트를 통과시키기 위한 조건 완화·단언 삭제·약화 금지. 연속성 검사를 약화시켜 생성을 깔끔해 보이게 만들지 않는다 — 충돌은 드러낸다.
3. **상태 문서 조작 금지.** progress.md·session-handoff.md는 세션 종료 절차(§6)에 따라 갱신하는 것 외에 임의 수정 금지.
4. **공용 헬퍼 복사 금지** — `src/lib`에서 import한다. 이동 슬라이스면 옮긴 본문은 byte-identical, 이동 심볼의 테스트 단언은 정의 파일로 재배치(삭제 금지).
5. **[onboard-mirror] 핀** — `buildOnboardChatPrompt`의 JSON 계약·응결 조건·condense 지시 3줄은 `promptBuilders.ts`↔`tools/storyx.mjs` byte-identical. 프롬프트 수정 시 두 곳 동시 + 핀 테스트.
6. **디자인 토큰** — 랜딩 `--lc-*`·브리지 `--nx-*` 유지, 스튜디오는 `--st-*`가 값 원천(`--sx-*`는 매핑). 새 transition/animation은 `--st-dur-*`/`--st-ease`만. 홈(hx-*) 신규 버튼은 `color: var(--nx-ink)` 명시(UA 검정 상속 비가시 이력).
7. **TDD 순서** — 새 생성 동작은 `src/lib/storyEngine.test.ts`(또는 해당 모듈 테스트)를 먼저 고친다.
8. **에러는 실제 로그를 읽고 고친다** — 패턴 매칭으로 "흔한 수정"을 적용하지 않는다. 에러 판정 휴리스틱은 실 provider 1회 실측으로 검증한다(codex exec는 정상 성공에도 stderr에 배너를 쓴다 — stderr 비어있지-않음 ≠ 에러).
9. **범위** — 한 번에 한 슬라이스. 활성 슬라이스의 파일 목록 밖은 손대지 않는다. session-handoff.md 최신 노트의 "손대지 말 것" 준수(OnboardingDraft 통합 영속·매체 변경 클리어 ref·playSeedEntry 분기·isHomeFlowStep 등).

## 6. 세션 종료 절차 (매 세션, 순서대로)

1. `bash init.sh` 재실행, 결과를 progress.md "최근 검증"에 기록.
2. progress.md 활성 작업 절 갱신(증거 = 커밋 SHA·파일 경로·캡처 경로).
3. session-handoff.md 맨 위에 새 인계 노트(파일 하단 템플릿 사용).
4. 활성 브랜치에 커밋(제목에 슬라이스 id). **main 직접 커밋 금지 — 브랜치→PR.** 머지는 사용자 몫.

## 7. 검증 계약

Codex의 자기보고는 완료 근거로 인정되지 않는다. 별도 Claude 세션이 다음을 독립 검증한 뒤에만 머지된다 — (1) tsc·test·build 재실행 (2) `git status`·파일 mtime으로 실반영 확인 (3) 우회주석 스캔 (4) 상태문서 diff 확인 (5) 헬퍼 중복·순환 import 점검 (6) `git diff main` 기준 리뷰(작업 중 main이 움직였으면 충돌·결측 함수 예상). 이 검증을 전제로 작업하라 — 검증을 통과할 수 없는 지름길은 처음부터 이득이 없다.
