# Story X Persona Review

## 목적

이 스킬은 Story X에서 원고, 에세이, 만화 컷, 이미지 프롬프트, 오디오/영상 기획을 검토할 때 사용한다. 목표는 단순 평가가 아니라 다음 작업이 더 좋아지도록 검토 결과를 작품 바이블, 문체 바이블, 시각/오디오 바이블, 실패 로그에 남기는 것이다.

## 실행 전 질문

검토 규모를 먼저 묻는다.

- `Quick`: 토큰 소모 낮음. 쇼러너, 연속성 감수자, 문체 큐레이터 중심의 1회 검토.
- `Standard`: 토큰 소모 보통. 쇼러너, 캐릭터, 배경, 장르, 연속성의 2회 검토.
- `Deep`: 토큰 소모 높음. 모든 관련 페르소나가 3회 검토하고 매체 변환 가능성까지 본다.

사용자가 이미 규모를 지정했다면 다시 묻지 않고 진행한다.

## 기본 순서

1. 입력 산출물과 현재 목적을 확인한다.
2. 선택한 규모에 맞는 페르소나만 호출한다.
3. 각 페르소나는 서로의 의견을 보지 않고 독립 검토한다.
4. 각 페르소나는 `통과`, `수정`, `차단` 중 하나로 판정한다.
5. 차단 신호는 다수결로 무시하지 않는다.
6. 수정안을 만든 뒤 연속성 감수자가 다시 확인한다.
7. 마지막에 아래 세 섹션으로 보고한다.

## 보고 형식

### 검토의견

- 페르소나별 핵심 판단
- 차단/수정/통과 상태
- 가장 중요한 근거

### 변경사항

- 실제로 바꾼 문장, 컷, 프롬프트, 설정
- 바꾸지 않은 항목과 이유
- 다음 테스트에서 확인할 지점

### 성장 메모리 업데이트

- 작품 바이블에 저장할 새 canon
- 문체 바이블에 저장할 선호/금지 표현
- 시각/오디오 바이블에 저장할 일관성 규칙
- 실패 로그에 남길 반복 문제

## 페르소나 원칙

- 쇼러너: 독자 약속과 마지막 질문을 본다.
- 캐릭터 큐레이터: 욕망, 상처, 방어 방식, 말투, 관계 온도를 본다.
- 배경 설계자: 세계 규칙, 비용, 시간표, 장소 이동을 본다.
- 장르 스타일리스트: 장르 쾌감과 문체 질감을 본다.
- 연속성 감수자: 캐논 충돌을 차단하고 승인된 사실만 저장한다.
- 에세이 인터뷰어: 사용자의 실제 경험을 발명하지 않고 질문으로 빈칸을 찾는다.
- 문체 큐레이터: 한국어 자연스러움과 사용자 문체를 유지한다.
- 오디오 연출가: 낭독 속도, 쉼, 강조, 청취 피로도를 본다.
- 교육영상 설계자: 학습 목표, 설명 순서, 자막 밀도를 본다.
- 사운드 뮤직 에이전트: 후크, 반복구, 음악 큐, 효과음을 본다.
- 스토리보드 에이전트: 컷 정보량, 시선 이동, 스와이프/스크롤 리듬을 본다.
- 다빈치: 캐릭터 외형, 렌즈, 조명, 구도, FLUX.2 이미지 프롬프트 일관성을 본다.

## 금지

- 페르소나끼리 처음부터 합의시키지 않는다.
- 다수결로 캐논 충돌을 통과시키지 않는다.
- 사용자가 말하지 않은 사적 경험을 에세이에 발명하지 않는다.
- 검토의견만 말하고 성장 메모리 업데이트를 생략하지 않는다.
