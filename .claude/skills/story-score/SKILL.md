# StoryScore v0.2 — 공식 채점 절차

이 스킬은 Story X 작품(백업 JSON 또는 chNN.md 디렉토리)의 완성도를 `STORYSCORE_V0_2` 공식으로 채점할 때 사용한다. 코드로 잴 수 있는 것은 CLI가 재현 가능하게 재고, 의미 판단은 아래 루브릭으로 심사한다. 공식·루브릭이 바뀌면 버전을 올리고 과거 점수와 섞지 않는다.

## 점수 공식 (100점)

| 축 | 배점 | 측정 방식 |
|---|---|---|
| 연속성 | 30 | 심사(루브릭 1~5 → 환산) + 결정론 보조(로스터 출현 매트릭스·철자 변형 의심 감점) |
| 페이오프/구조 | 25 | 심사 — 떡밥 대차(발급/회수 목록 추출 → 회수율), 결말의 전제 이행 |
| 프로즈 품질 | 20 | 심사 — 표본 회차 루브릭(문체 일관·장면 구체성·대사·과설명) |
| 연재성/후크 | 15 | 결정론 후크 휴리스틱(7) + 심사(8) |
| 형식 규율 | 10 | 결정론 — 회차 분량 분포(목표 1,800~2,700자 적합률) |

- 심사 축 환산 — `환산점 = 배점 × (루브릭 점수 / 5)`. 루브릭은 1~5 정수.
- **근거 인용 의무** — 모든 심사 점수에는 회차 번호와 실제 문장 인용이 붙어야 한다. 인용 없는 점수는 무효이며 그 축을 재심사한다.
- 하네스 메타데이터(canonFacts·stakesLedger 개수)는 점수에 절대 쓰지 않는다. 입력 비대칭(통제군에 없음) 때문이며, 부가 진단으로만 언급한다.

## 절차

1. **CLI 실행** — `node tools/storyscore.mjs --input <백업.json|md디렉토리> --roster "이름1,이름2" --out report.json`
2. **결정론 점수 채택** — `report.json` 의 `deterministic.scores` 를 그대로 쓴다. 형식 규율(10)과 후크 결정론(7)은 다시 계산하거나 보정하지 않는다.
3. **judgePackage 심사** — `judgePackage.samples`(1화·25%·50%·75%·최종 전문)를 깊이 읽고, `judgePackage.sequence`(전 회차 제목·말미 200자)로 전체 흐름을 훑은 뒤 아래 4개 축을 루브릭으로 채점한다. 각 축마다 점수(1~5)·근거 인용(회차+문장)·한 단락 판단을 적는다.
4. **결정론 보조 반영(연속성 축)** — `deterministic.roster.variantSuspicions` 에 철자 변형 의심이 있으면 실제 오기인지 표본에서 확인하고, 오기로 확인된 건당 연속성 환산점에서 1점을 감점한다(최대 -3, 0점 미만 불가). 의도된 별칭·말장난으로 확인되면 감점하지 않고 그 판단 근거를 적는다.
5. **합산** — 결정론(형식 규율 + 후크 결정론) + 심사 환산(연속성 + 페이오프 + 프로즈 + 연재성 심사분)을 더해 100점 만점 총점을 만든다.
6. **스코어카드 출력** — 아래 출력 형식으로 마크다운 스코어카드를 작성한다.

## 심사 루브릭 (4개 축)

### 1. 연속성 — 30점 (루브릭 1~5 → ×6)

인물·세계 규칙·사실이 회차를 가로질러 어긋나지 않는가. **후반 표본(75%·최종화)을 가중해서 본다** — 단일 컨텍스트로 쓴 작품은 초반 연속성이 자연히 유리하므로, 초반이 매끈해도 후반 표본에서 초반 설정(인물의 상처·세계 규칙·지명·관계)이 살아 있는지가 점수를 가른다. 75%·최종화 표본에서 발견한 어긋남은 같은 무게의 초반 어긋남보다 한 단계 낮게 평가한다.

- **1점** — 표본 안에서 명백한 모순이 반복된다. 인물 이름·관계·이미 죽거나 떠난 인물의 재등장, 세계 규칙의 정면 위반이 두 건 이상이고, 후반 표본이 초반 설정을 사실상 잊었다.
- **3점** — 굵은 사실(인물·생사·핵심 규칙)은 유지되지만 잔금이 보인다. 시간 경과·소지품·부상 같은 세부가 한두 군데 어긋나거나, 초반에 세운 설정 일부가 후반에서 언급 없이 증발한다.
- **5점** — 표본과 말미 시퀀스 전체에서 모순을 찾지 못했고, 후반 표본이 초반의 구체 설정(상처·규칙·약속)을 정확히 기억해 되살린다. 어긋남처럼 보이는 대목이 있다면 작품 안에서 설명된다.

근거 인용 — 유지/위반의 실제 문장을 초반 표본과 후반 표본에서 각각 최소 1건씩 짝지어 인용한다.

### 2. 페이오프/구조 — 25점 (루브릭 1~5 → ×5)

작품이 건 약속(떡밥·질문·위협)을 실제로 회수하는가. 표본과 말미 시퀀스에서 **발급된 약속 목록과 회수된 약속 목록을 먼저 추출**하고, 회수율과 회수의 질(우연이 아니라 인물의 선택·대가로 회수되는가)을 본다. 최종화가 작품의 전제(표면 약속과 심층 질문)에 답하는지가 핵심이다.

- **1점** — 굵은 떡밥 대부분이 회수되지 않거나, 결말이 전제와 무관한 사건으로 끝난다. 회수가 있어도 우연·고백·설명으로 처리된다.
- **3점** — 중심 약속은 회수되지만 일부 떡밥이 증발하고, 회수 일부가 편의적(때마침 나타난 인물·갑작스런 능력)이다. 결말이 전제에 답하긴 하나 비용 없이 답한다.
- **5점** — 추출한 약속 목록의 핵심이 모두 회수되고, 회수가 인물의 선택과 대가를 통과한다. 최종화가 1화의 약속을 정면으로 이행하며, 미회수 떡밥은 의도된 여운으로 읽힌다.

근거 인용 — 발급 문장(앞 표본)과 회수 문장(뒤 표본)을 짝지어 인용한다. 증발한 떡밥은 발급 문장만 인용하고 미회수로 표기한다.

### 3. 프로즈 품질 — 20점 (루브릭 1~5 → ×4)

표본 회차의 문장 자체. 문체 일관성, 장면의 구체성(추상 감정어 대신 사물·동작·감각), 대사의 인물 분화, 과설명 여부를 본다.

- **1점** — 표본 사이에 문체가 흔들리고(경어·시제·시점 혼선), 장면이 요약·설명으로 처리되며, 대사가 누구의 입에서 나와도 같다. 감정과 의미를 서술자가 직접 해설한다.
- **3점** — 문체는 대체로 일관되고 장면도 그려지지만, 군데군데 과설명·상투 표현·번역투가 끼고, 일부 조연의 대사가 분화되지 않는다.
- **5점** — 다섯 표본의 문체가 한 작가의 것으로 읽히고, 장면이 구체 감각으로 서고, 주요 인물의 대사만 읽어도 화자를 구분할 수 있다. 의미를 강요하지 않고 독자가 채울 여백을 남긴다.

**v0.2 — 온건함(tameness) 감점.** 문장이 매끄럽고 모순이 없어도 *날것·긴장이 없으면* 매력이 떨어진다(웹소설 식상함의 원인). 다음이 표본에 반복되면 프로즈 점수를 한 단계 낮춘다 — ① 갈등이 같은 장면 안에서 즉시 봉합된다 ② 인물이 손해 보는 선택 없이 모두 합리적으로 행동한다 ③ 서술자가 감정·의미를 직접 해설한다("그는 더 이상 두렵지 않았다" 류) ④ 위험·욕망의 간극이 작아 다음을 궁금하게 만들지 않는다. 감점 시 근거 문장을 인용한다.

근거 인용 — 잘된 문장과 문제 문장을 각각 회차 번호와 함께 인용한다.

### 4. 연재성/후크 — 심사분 8점 (루브릭 1~5 → ×1.6)

결정론 후크 점수(7점)는 CLI가 이미 쟀다. 심사분은 후크의 **질**을 본다 — 말미가 단순히 물음표로 끝나는 게 아니라, 다음 회차를 읽지 않을 수 없는 압력(새 정보·역전·위협의 구체화)을 만드는가. 말미 시퀀스(전 회차 tail200)를 훑어 후크의 다양성과 누적 설계도 본다.

- **1점** — 말미가 장면 종료·요약으로 풀려 끝나거나, 후크가 있어도 같은 패턴(누군가 나타났다 류)의 반복이다.
- **3점** — 회차 대부분이 다음 화 압력을 만들지만, 후크가 사건 제시에 머물고 인물의 선택·대가와 연결되지 않는 회차가 섞여 있다.
- **5점** — 말미마다 결이 다른 후크(정보 역전·관계 전환·시한 압박)가 걸리고, 후크가 그 회차 내용에서 자라나며, 다음 회차 표본이 실제로 그 후크를 이어받는다.

근거 인용 — 말미 시퀀스에서 잘 걸린 후크와 풀려 끝난 말미를 각각 회차 번호와 함께 인용한다.

**v0.2 — 제목 반복 감점(U1).** `deterministic.titles.repetitionRate` 가 0.5 이상이면 제목이 같은 템플릿으로 반복되는 신호다. 표본의 제목 시퀀스에서 실제로 같은 어간·구조가 반복되는지 확인하고, 확인되면 연재성 심사분을 한 단계(1점) 낮춘다. 의도된 연작 표제(시즌·부제 체계)로 확인되면 감점하지 않고 근거를 적는다.

## A/B 채점 시 라벨 중립화

두 작품을 비교 채점할 때는 심사자가 어느 쪽이 실험군인지 알 수 없게 한다.

- 입력 라벨을 `작품 A` / `작품 B` 로 중립화한다. 파일 경로·도구 이름·하네스 흔적(canonFacts 유무 등)을 심사 프롬프트에 드러내지 않는다.
- 양쪽에 같은 표본 규칙·같은 루브릭·같은 심사 모델을 1회씩 적용한다.
- 부가 진단(하네스 메타)은 채점이 모두 끝난 뒤에만 열어 본다.
- 통제군(단일 컨텍스트 집필)은 초반 연속성이 유리할 수 있으므로, 연속성 축의 후반 표본 가중 규칙을 양쪽에 동일하게 적용한다.

## 스코어카드 출력 형식

```markdown
# STORYSCORE_V0_2 — 총점 NN.N/100

| 축 | 배점 | 점수 | 근거 요약 |
|---|---|---|---|
| 연속성 | 30 | NN.N | 루브릭 N/5, 변형 감점 -N |
| 페이오프/구조 | 25 | NN.N | 루브릭 N/5, 회수율 N/N |
| 프로즈 품질 | 20 | NN.N | 루브릭 N/5 |
| 연재성/후크 | 15 | NN.N | 결정론 N.N/7 + 심사 N.N/8 |
| 형식 규율 | 10 | NN.N | 적합률 NN% |

## 축별 근거 인용
(축마다 회차 번호 + 실제 문장 인용 + 한 단락 판단)

## 결정론 진단 부록
(분량 분포·후크 매트릭스·로스터 매트릭스·철자 변형 의심·하네스 부가 진단)
```

## 금지

- 근거 인용 없이 심사 점수를 적지 않는다.
- 결정론 점수를 심사자가 임의로 보정하지 않는다.
- 하네스 메타데이터를 점수에 반영하지 않는다.
- 한 작품의 심사 결과를 본 뒤 다른 작품의 루브릭 기준을 바꾸지 않는다.
